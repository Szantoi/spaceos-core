## Status Update

BUG — TradeType enum parse — **CLOSED_DONE (misdiagnosed)**

### Investigation

The `JsonStringEnumConverter` works correctly for both input and output:

| Test | Result |
|------|--------|
| `{"tradeType":"Joinery","isExternalNode":false,"intentDataJson":"{}"}` | 201 Created |
| `{"tradeType":"Plumbing","isExternalNode":false,"intentDataJson":"{}"}` | 201 Created |
| `{"status":"Active"}` (work-station) | 200 OK |

### Root Cause of Original 500

The original curl was missing the `intentDataJson` field:
```bash
curl -d '{"tradeType":"Joinery","isExternalNode":false}'  # ← no intentDataJson!
```

When `isExternalNode=false`, the handler calls `SpaceLayer.CreateLocalLayer(request.IntentDataJson!, ...)` which null-references because `IntentDataJson` is null. The `RegisterSpaceLayerCommandValidator` has a `NotEmpty()` rule for this field, but the request with null `IntentDataJson` should be caught as a 422 before reaching the handler.

This is a validation pipeline edge case — the validator's `When(!x.IsExternalNode)` + `NotEmpty()` should catch null, but it's reaching the handler. This may be a FluentValidation null-handling issue worth investigating separately.

### No enum-related code changes required

The `JsonStringEnumConverter` added in MSG-006 works correctly for both serialization (output) and deserialization (input).
