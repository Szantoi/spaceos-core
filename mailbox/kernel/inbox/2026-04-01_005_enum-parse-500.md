## Tárgy

BUG — TradeType enum parse error a POST space-layers endpointon (500)

## Probléma

```bash
curl -X POST "/api/facilities/:id/space-layers" \
  -d '{"tradeType":"Joinery","isExternalNode":false}'
# → 500: "The JSON value could not be converted... Path: $.tradeType"
```

Sem PascalCase (`Joinery`) sem camelCase (`joinery`) nem működik.

A `JsonStringEnumConverter` a `ConfigureHttpJsonOptions`-ben van beállítva, de a Minimal API `RegisterSpaceLayerRequest` record-jában a `TradeType` property közvetlenül a domain enumot használja — lehet, hogy a serializer nem alkalmazza rá az enum converter-t.

## Elvárt megoldás

A `JsonStringEnumConverter`-nek case-insensitive módban kell működnie, és minden enum property-ra alkalmazódnia kell:

```csharp
new JsonStringEnumConverter(JsonNamingPolicy.CamelCase)
// vagy
new JsonStringEnumConverter(allowIntegerValues: true)
```

Teszteld ezeket a hívásokat:
```bash
curl -d '{"tradeType":"Joinery","isExternalNode":false}' → 201
curl -d '{"status":"Active"}' → 200  (work-station status)
```
