## Ellenőrzött Eszközök

### ✅ 1. React Hook Generator

**Skill:** `/react-hook-generator`
**Lokáció:** `.claude/skills/react-hook-generator/SKILL.md` (201 sor)
**MCP Tool:** `generate_hook`

**Támogatott hook típusok:**
1. **Query Hook** — TanStack Query useQuery (data fetching)
   - `--with-cache` flag: 5 min staleTime
   - Auto-generated queryKey

2. **Mutation Hook** — TanStack Query useMutation (data modification)
   - Auto invalidation queryClient alapján
   - onSuccess callback beépítve

3. **State Hook** — useState + useCallback
   - State management pattern (step, data)
   - Reset, setStep, setData callbacks

4. **Effect Hook** — useEffect + cleanup
   - Side effect patterns (pl. window resize listener)
   - SSR-safe initial state

**Generált fájlok:**
```
client/src/hooks/
  └── use<Name>/
      ├── use<Name>.ts       # Fő hook
      ├── use<Name>.test.ts  # Teszt (--with-test)
      └── index.ts           # Export
```

**CLI példa:**
```bash
spaceos generate hook Orders --type query --with-cache --endpoint /api/orders
```

**ROI:** 80% időmegtakarítás boilerplate kódon

### ✅ 2. FSM Subscription System

**Skill:** `/fsm-subscription`
**Lokáció:** `.claude/skills/fsm-subscription/SKILL.md` (146 sor)

**MCP Tools:**
1. `subscribe_to_task` — Task állapotváltozás (done/blocked/progress)
2. `subscribe_to_terminal` — Terminal események (inbox/outbox/session)
3. `unsubscribe` — Feliratkozás törlés
4. `get_subscriptions` — Aktív feliratkozások listázása

**Delivery methods:**
- **SSE** — Aktív session (<100ms latency) ← Preferált
- **Telegram** — Nincs aktív session (1-2s)
- **Inbox** — Fallback, garantált delivery

**SSE endpoint:**
```bash
curl -N "http://localhost:3456/api/subscriptions/events?terminal=frontend"
```

**Events:**
- Task: `done`, `blocked`, `progress`
- Terminal: `inbox_new`, `outbox_done`, `session_started`, `session_ended`

**Expiration:** 1 óra default, auto-cleanup 60s intervallumonként

### FSM Subscription — Coordination Examples

**Scenario:** Frontend vár Backend API-ra
```typescript
// Feliratkozás Backend task DONE-jára
await mcp.subscribe_to_task({
  terminal: "frontend",
  task_id: "MSG-BACKEND-097",
  events: ["done"],
  delivery_method: "sse"
});

// SSE listener
const es = new EventSource("/api/subscriptions/events?terminal=frontend");
es.onmessage = (e) => {
  const data = JSON.parse(e.data);
  if (data.type === "task" && data.event.name === "outbox:done") {
    console.log("Backend API ready! Start integration testing.");
  }
};
```

**ROI:** Polling helyett push (0 wasted API calls, <100ms notification)
