# CRITICAL SECURITY: REST /api/mailbox Authentication Required

## Problem

A Cabinet-VPS biztonsági audit feltárta: a `/api/mailbox/*` REST végpontok **token nélkül hívhatók**.

```bash
# Ez NEM kellene működjön auth nélkül:
curl http://localhost:3456/api/mailbox/root/inbox  # HTTP 200 ← HIBÁS
```

Az MCP `/mcp/` tokenköteles, de a REST réteg megkerüli az agent-identitás modellt.

## Impact

- Bárki olvashat/írhat bármely terminál inbox/outboxába
- Cabinet-VPS-en 2026-07-07-én incidens történt (MSG-EXPLORER-051/052 szemét)

## Required Fix

Implementálj két middleware-t a `src/mcp.ts`-be:

1. **`authenticateRest`**: Bearer token kötelező, 401/403, `req.mcpTerminal` beállítás
2. **`authorizeMailboxRest`**: 
   - root/conductor: mindent
   - monitor: GET-ek
   - más terminál: CSAK saját mailbox
   - idegen inboxba POST: tool-permissions szerint
   - DENY-k console.warn-nal naplózva

Bekötés: `app.use('/api/mailbox', authenticateRest, authorizeMailboxRest, mailboxRoutes)`

## Test Matrix (8 eset)

| Case | Expected |
|------|----------|
| No token | 401 |
| Invalid token | 403 |
| Valid token, own mailbox | 200 |
| Valid token, other inbox (non-coordinator) | 403 |
| Coordinator, any mailbox | 200 |
| Monitor, GET any | 200 |
| Monitor, POST other | 403 |
| Broadcast (coordinator only) | 200/403 |
