# Alert Rules Implementation

## Kontextus

Automatikus alertek szükségesek kritikus állapotok detektálására az autonóm fejlesztési folyamatokban. A rendszernek proaktívan kell jelezni a problémákat.

## Feladat

Implementálj egy alert rendszert, amely 4 szabály alapján Telegram értesítést küld kritikus állapotokra.

## Alert szabályok

| Trigger | Küszöb | Értesítés formátum |
|---------|--------|-------------------|
| **Session stuck** | >30 min no activity | `🔴 [ALERT] {terminal} stuck >30min` |
| **Consecutive skips** | 3+ autonomous skip | `⚠️ [WARNING] Túl sok párhuzamos munka (3+ skip)` |
| **BLOCKED timeout** | >2h unresolved | `🟡 [ESCALATION] {task} blocked >2h` |
| **No activity** | >2h no task | `ℹ️ [INFO] Nincs aktív fejlesztés >2h` |

## Specifikáció

### Implementáció részletek

**Új fájl:** `spaceos-nexus/knowledge-service/src/pipeline/alertRules.ts`

**Integráció:**
- `src/pipeline/nightwatch.ts` hívja minden ciklusban (2 percenként)
- State tracking: `.alertState.json` fájl vagy SQLite tábla

**Adatforrások:**
- Session activity: `GET /api/sessions/all`
- Autonomous status: `GET /api/autonomous/status`
- BLOCKED tasks: fájl scan `terminals/*/outbox/` mappa

### Komponensek

1. **alertRules.ts:**
   ```typescript
   export interface AlertRule {
     id: string;
     trigger: 'session_stuck' | 'consecutive_skips' | 'blocked_timeout' | 'no_activity';
     threshold: number; // minutes
     cooldown: number; // minutes, spam prevention
   }

   export interface AlertState {
     lastFired: Record<string, Date>;
   }

   export async function checkSessionStuck(): Promise<string | null>
   export async function checkConsecutiveSkips(): Promise<string | null>
   export async function checkBlockedTimeout(): Promise<string | null>
   export async function checkNoActivity(): Promise<string | null>
   export async function runAlertRules(): Promise<void>
   ```

2. **State management:**
   - Fájl: `/opt/spaceos/.alertState.json`
   - Struktúra: `{ "lastFired": { "session_stuck_backend": "2026-06-22T10:00:00Z", ... } }`
   - Cooldown: minden alert típus max 1× / 30 perc

3. **Telegram notification:**
   - Használd a meglévő `telegram()` helper-t
   - Formátum: emoji prefix + message

4. **Unit testek:** `src/pipeline/__tests__/alertRules.test.ts`

## Részletes logika

### 1. Session Stuck (>30 min)
```typescript
// Lekérdezés: GET /api/sessions/all
// Ellenőrzés: session.lastActivity < (now - 30min) && session.active === true
// Alert: 🔴 [ALERT] {terminal} stuck >30min
```

### 2. Consecutive Skips (3+)
```typescript
// Lekérdezés: GET /api/autonomous/status
// Ellenőrzés: status.recentSkips >= 3
// Alert: ⚠️ [WARNING] Túl sok párhuzamos munka (3+ skip)
```

### 3. BLOCKED Timeout (>2h)
```typescript
// Fájl scan: terminals/*/outbox/*.md
// Frontmatter filter: type === 'blocked' && status === 'UNREAD'
// Ellenőrzés: created < (now - 2h)
// Alert: 🟡 [ESCALATION] {task} blocked >2h
```

### 4. No Activity (>2h)
```typescript
// Lekérdezés: GET /api/sessions/all
// Ellenőrzés: max(session.lastActivity) < (now - 2h)
// Alert: ℹ️ [INFO] Nincs aktív fejlesztés >2h
```
