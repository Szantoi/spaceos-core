## Key Learning Points (Pattern Mastery)

### 1. **3-Param Repository Pattern** ✅
- **Different from HR/Maintenance:** QA uses ONLY 3-param pattern (explicit tenantId on every method)
- **Not hybrid:** No implicit tenant context from DbContext
- **Benefit:** Explicit, testable, audit-trail ready
- **Validation:** Test scenario #5 proves isolation works correctly

### 2. **Owned Collection Mapping** ✅
- **InspectionCriteria:** Maps to `qa_checkpoint_criteria` table, separate from parent
- **FailureNote:** Maps to `inspection_defects` table, separate from parent
- **Fixed issue from Maintenance:** All properties have explicit `.HasColumnName()` to prevent EF Core naming mismatches
- **Composite keys:** EF Core auto-manages via `new { parent_id, id }` pattern

### 3. **Domain Enum Validation** ✅
- **CheckpointType:** Incoming, InProcess, Final (NOT Visual/Dimensional/Assembly)
- **CriticalLevel:** Critical, Major, Minor (NOT High/Low)
- **CriteriaType:** Visual, Dimensional, Functional (flexible)
- **FailureType:** Scratch, Gap, Misalignment, Color, Dimension, Surface, Functional, Missing, Damage, Other
- **InspectionStatus:** Planned, InProgress, Completed
- **InspectionResult:** Pending, Pass, Fail
- **Learning:** Always validate enum values before writing tests

### 4. **Migration Generation & Folder Placement** ✅
- **Issue encountered:** `dotnet-ef` created `src/src/Infrastructure/...` (double nesting)
- **Fix:** Used `--output-dir` parameter correctly; manually moved files to `src/Infrastructure/Persistence/Migrations/`
- **Lesson:** Be explicit with migration paths

### 5. **Testcontainers Setup** ✅
- **Auto-migration:** `await _context.Database.MigrateAsync()` in fixture InitializeAsync
- **Container lifecycle:** IAsyncLifetime handles StartAsync/DisposeAsync
- **Collection fixture:** xUnit collection definition for sharing container across tests
- **Performance:** PostgreSQL 16 Alpine is fast; tests can run in seconds
