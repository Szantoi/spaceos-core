#### B.2 — buildStartContext() Implementation ✅

**Algorithm:**
1. Query hot memories (terminal-specific, limit 10)
2. Query warm memories (terminal-specific, limit 5)
3. Query shared memories (global, limit 3)
4. Build markdown context with emoji headers:
   - 🔥 Hot Memories (Recent, High Priority)
   - 🌤️ Warm Memories (Past 2 weeks)
   - 🌍 Shared Memories (Cross-Terminal)
5. Estimate token count (~4 chars per token)
6. Return SessionStartResult

**Features:**
- Max context size: ~5K tokens (configurable)
- Respects terminal scoping (hot/warm per terminal, shared global)
- Structured markdown for readability
- Emoji-based memory type indicators

#### B.3 — handleSessionEnd() Implementation ✅

**Algorithm:**
1. Insert session metadata into `session_history` table
2. Auto-save session summary as hot memory (if summary provided)
3. Trigger retrospective if:
   - `hadCorrections === true` OR
   - `toolCallCount > 10`
4. Placeholder for handoff generation (future work)
5. Return SessionEndResult

**Features:**
- Captures end_reason: 'done' | 'blocked' | 'timeout' | 'handoff' | 'crash'
- Tracks tool_calls and had_corrections for retrospective decision
- Auto-saves summaries to hot tier for next session

#### B.4 — Integration into sessionStarter.ts ✅

**Modified:** `startTerminalSession()` function

**Cold start flow:**
```typescript
// 1. Create tmux session
await execAsync(`tmux new-session -d -s ${sessionName} ...`);

// 2. Build startup context (BEFORE claude command)
const startContext = await buildStartContext({
  terminal,
  inboxMessageId: messageId,
  taskId: messageId,
});

// 3. Start claude
await execAsync(`tmux send-keys -t ${sessionName} "claude --model ${model}" Enter`);

// 4. Wait for claude to start (2s)
await new Promise(resolve => setTimeout(resolve, 2000));

// 5. Inject context into session
if (startContext.memoriesLoaded > 0) {
  injectMessageToSession(sessionName, startContext.contextMarkdown);
}
```

**Key changes:**
- Context injection happens AFTER claude starts but BEFORE user interaction
- Logs memory count and token estimate
- Gracefully handles no-memory case (skip injection)

#### B.5 — session_history Table Created ✅

  terminal TEXT NOT NULL,
  started_at TEXT NOT NULL,
  ended_at TEXT,

  memories_created INTEGER NOT NULL DEFAULT 0,
  tool_calls INTEGER NOT NULL DEFAULT 0,
  had_corrections BOOLEAN NOT NULL DEFAULT 0,
  retrospective_done BOOLEAN NOT NULL DEFAULT 0

CREATE INDEX idx_session_history_terminal ON session_history(terminal);
CREATE INDEX idx_session_history_started ON session_history(started_at DESC);
