## ADR-046 Track A: Memory Tier Management — COMPLETE

### Summary

Successfully implemented **Memory Tier Management** for the Nexus Knowledge Service as specified in ADR-046. The system now supports 4-tier memory semantics (hot/warm/cold/shared) with tier-specific decay rates and promotion logic.

---

### Implementation Details

#### A.1 — Tier Columns Added ✅

**File:** `src/pipeline/memoryStore.ts` (initSchema)

```sql
ALTER TABLE memories ADD COLUMN tier TEXT NOT NULL DEFAULT 'hot'
  CHECK (tier IN ('hot', 'warm', 'cold', 'shared'));
ALTER TABLE memories ADD COLUMN promoted_from TEXT;
ALTER TABLE memories ADD COLUMN promotion_count INTEGER NOT NULL DEFAULT 0;
ALTER TABLE memories ADD COLUMN last_promotion_at TEXT;
```

**Added indexes:**
- `idx_memories_tier` — for tier-based queries

#### A.2 — Tier-Aware Save/Query ✅

**New exports:**
- `saveTieredMemory(input: TieredMemoryInput): Promise<TieredMemory>`
- `queryByTier(terminal: string, tiers: MemoryTier[], limit?: number): TieredMemory[]`
- `type MemoryTier = 'hot' | 'warm' | 'cold' | 'shared'`
- `interface TieredMemory extends Memory { tier, promotedFrom, promotionCount, lastPromotionAt }`

**Features:**
- Tier-aware INSERT with explicit tier assignment
- Queries support multiple tiers (e.g., `['hot', 'warm']`)
- Respects terminal scoping: shared tier visible to all terminals
- Orders by salience DESC, accessed_at DESC

#### A.3 — Tier Promotion Logic ✅

**New export:**
- `promoteMemory(id: number, newTier: MemoryTier, reason: string): Promise<void>`

**Features:**
- Tracks promotion history (promoted_from, promotion_count, last_promotion_at)
- Validates memory existence before promotion
- Logs promotion with reason

#### A.4 — Tier-Specific Decay Rates ✅

**Modified:** `runSalienceDecay()`

**Tier policies (from ADR-046):**
```typescript
TIER_POLICIES = {
  hot:    { decayRate: 0.15, maxAge: '48h' },   // 15% per day
  warm:   { decayRate: 0.05, maxAge: '14d' },   // 5% per day
  cold:   { decayRate: 0.01, maxAge: '365d' },  // 1% per day
  shared: { decayRate: 0,    maxAge: 'forever' } // no decay
}
```

**Implementation:**
- Iterates over TIER_POLICIES and applies decay per tier
- Skips tiers with `decayRate === 0` (shared)
- Logs total affected memories

#### A.5 — Shared Memories Table ✅

**New tables:**
```sql
CREATE TABLE shared_memories (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  content TEXT NOT NULL,
  keywords TEXT NOT NULL DEFAULT '',
  context TEXT,
  salience REAL NOT NULL DEFAULT 0.7,
  created_by TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  accessed_at TEXT NOT NULL DEFAULT (datetime('now')),
  access_count INTEGER NOT NULL DEFAULT 0
);

CREATE VIRTUAL TABLE shared_memories_fts USING fts5(
  content, keywords,
  content='shared_memories',
  content_rowid='id'
);
```

**FTS5 triggers:** INSERT/DELETE/UPDATE triggers to keep FTS in sync
