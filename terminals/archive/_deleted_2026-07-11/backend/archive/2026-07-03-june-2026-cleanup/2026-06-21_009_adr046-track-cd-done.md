### Track C Implementation Details

#### C.1 — retrospective.ts Module Created ✅

**File:** `src/retrospective.ts` (422 lines)

**Key exports:**
```typescript
export async function runRetrospective(input: RetrospectiveAnalysisInput): Promise<RetrospectiveResult>
export async function applyRetrospective(input: ApplyRetrospectiveInput): Promise<ApplyRetrospectiveResult>
export function closeRetrospective(): void
```

**Type definitions:**
- `RetrospectiveProposal` — skill/memory/workflow improvement proposals
- `RetrospectiveAnalysisInput` — scope (session/last-task/last-hour), focus (skills/memory/workflow/all)
- `ApplyRetrospectiveResult` — execution results (skills created, memories saved, workflows updated)

**Features:**
- Analyzes session history from `session_history` table
- Generates 3 types of proposals: skills, memory, workflow
- Heuristic-based proposal generation (corrections → skill, success → memory, high tool calls → workflow)
- Executes approved proposals (create skills, save/retier memories)
- Tracks proposal history in `retrospective_proposals` table

#### C.2 — handoff.ts Module Created ✅

**File:** `src/handoff.ts` (246 lines)

**Key exports:**
```typescript
export async function generateHandoff(input: GenerateHandoffInput): Promise<GenerateHandoffResult>
```

**Type definitions:**
- `HandoffDocument` — structured handoff with purpose, goal, progress, successes, failures, next steps
- `GenerateHandoffInput` — terminal, purpose, target, output (file/message)
- `GenerateHandoffResult` — success status, file path, document, markdown

**Features:**
- Queries hot+warm memories for context
- Extracts progress, successes, failures from memory content via regex
- Generates markdown HANDOFF.md document
- Supports both file output and message output
- Saves to `/opt/spaceos/terminals/{terminal}/handoff/HANDOFF_{date}_{purpose}.md`

#### C.3 — retrospective_proposals Table Created ✅

**SQL schema:**
```sql
CREATE TABLE retrospective_proposals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  terminal TEXT NOT NULL,
  session_id INTEGER,
  type TEXT NOT NULL CHECK (type IN ('skill', 'memory', 'workflow')),
  action TEXT NOT NULL CHECK (action IN ('create', 'patch', 'delete', 'save', 'retier')),
  target TEXT NOT NULL,
  reason TEXT NOT NULL,
  content TEXT,
  new_tier TEXT,
  priority TEXT NOT NULL DEFAULT 'medium' CHECK (priority IN ('high', 'medium', 'low')),
  approved BOOLEAN NOT NULL DEFAULT 0,
  executed BOOLEAN NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

**Indexes:**
- `idx_retrospective_terminal` — terminal scoping
- `idx_retrospective_session` — session correlation
- `idx_retrospective_approved` — approval filtering

#### D.1 — digest.ts Module Created ✅

**Key exports:**
```typescript
export async function generateDailyDigest(input: DailyDigestInput): Promise<DailyDigestResult>

```

**Type definitions:**
- `DailyDigestInput` — terminal, date (YYYY-MM-DD)
- `DailyDigestResult` — session count, memories created, tool calls, tasks completed/blocked, summary, markdown

**Features:**
- Queries `session_history` table for daily sessions
- Aggregates statistics (session count, tool calls, completion/blocked counts)
- Queries memories created on the date
- Generates markdown digest document
- Auto-saves digest as warm memory if sessions > 0

#### D.2 — REST API Endpoints (8 endpoints) ✅

**Memory Tier Management:**

1. **GET /api/memories/tiered** — Query memories by tier
   - Query params: terminal, tiers (comma-separated), limit
   - Returns: memories array with tier, type, content preview, salience

2. **POST /api/memories/save** — Save tiered memory
   - Body: tier, type, source, content, terminal, context, salience
   - Returns: memory ID, tier, type, salience

3. **POST /api/memories/:id/promote** — Promote memory to higher tier
   - Body: newTier, reason
   - Returns: success status, promotion metadata

**Session Lifecycle:**

4. **POST /api/session/start-context** — Get cold start context
   - Body: terminal, taskId
   - Returns: memories loaded, hot/warm/shared counts, context markdown, token estimate

5. **POST /api/session/end** — Handle session end
   - Body: terminal, endReason, taskId, summary, hadCorrections, toolCallCount
   - Returns: session ID, memories saved, retrospective triggered status

6. **GET /api/session/history** — Get session history
   - Query params: terminal (optional), limit (default: 10)
   - Returns: session history array from `session_history` table

**Daily Digest:**

7. **POST /api/digest/generate** — Generate daily digest
   - Body: terminal, date (optional, defaults to today)
   - Returns: digest statistics, summary, markdown, savedAsMemory flag

8. **GET /api/digest/:terminal/:date** — Get daily digest
   - Path params: terminal, date (YYYY-MM-DD)
   - Returns: digest statistics and markdown
