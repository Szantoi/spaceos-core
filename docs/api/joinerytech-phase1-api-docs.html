Architecture: .NET 8 REST API with PostgreSQL RLS, DDD/FSM, JWT+RBAC

  • Authentication (JWT HttpOnly Cookie)

Authentication

JWT-based authentication with HttpOnly cookies

User login

Authenticate user with email and password. Returns JWT token in HttpOnly cookie.

Security: HttpOnly cookie prevents XSS attacks (ADR-058) CSRF Protection: SameSite=Strict in production, SameSite=Lax in dev

Request Body schema: application/json
required
email
required
string <email>

User email address

password
required
string <password> >= 8 characters

User password (min 8 chars)

rememberMe
boolean
Default: false

Extend token expiration to 30 days

Responses

Production

https://api.joinerytech.hu/api/auth/login

Staging

https://api.staging.joinerytech.hu/api/auth/login

Development

http://localhost:5000/api/auth/login

Catalog

integer >= 1

pageSize
integer [ 10 .. 100 ]
Default: 50

string

Filter by category ID

string
Example: search=CABINET

Update lead status

Transition lead to a new status.

FSM: Valid transitions enforced by backend

  • new → contacted, lost
  • contacted → qualified, lost
  • qualified → converted, lost

RBAC: User must have 'crm.lead.update' permission

Authorizations:
cookieAuth
path Parameters
id
required
string
Example: lead-12345

Lead ID

Request Body schema: application/json
required
status
required
string
Enum: "new" "contacted" "qualified" "converted" "lost"
reason
string

Reason for status change (required for 'lost')
