## 1. Katalógus Képoptimalizálás — Radikális megközelítés

### Backend: Image CDN + WebP konverzió
```
POST /api/catalog/items/{id}/image/upload
→ S3 upload + CloudFlare Image Resizing aktiválás
→ Automatikus WebP/AVIF generálás
→ Response: { original, thumbnail, webp, avif }
```

**DB séma kiegészítés:**
```sql
ALTER TABLE CatalogItems ADD COLUMN image_variants JSONB;
-- { "thumb": "url", "webp": "url", "avif": "url", "blurhash": "L8F~..." }
```

### Frontend: Suspense-alapú progresszív betöltés
```jsx
// catalog-image.jsx — Új komponens
<Suspense fallback={<BlurhashCanvas hash={item.image_variants.blurhash} />}>
  <picture>
    <source srcSet={item.image_variants.avif} type="image/avif" />
    <source srcSet={item.image_variants.webp} type="image/webp" />
    <img loading="lazy" src={item.image_variants.thumb} />
  </picture>
</Suspense>
```

**Kockázat:** AVIF support 2024-ben még patchy Safari-ban → graceful degradation WebP-re.

## 2. Filter localStorage — Event-driven state sync

### Backend: NEM KELL
A szűrőlogika client-side marad, de előkészítjük a server-side filter preset sync-et:

```
GET /api/users/me/preferences
→ { catalog_filters: {...}, view_mode: "grid" }

PATCH /api/users/me/preferences
→ Body: { catalog_filters: {...} }
```

### Frontend: Zustand slice + broadcast channel
```javascript
// app-store.jsx
const useCatalogStore = create(
  persist(
    (set) => ({
      filters: {},
      setFilters: (filters) => {
        set({ filters });
        // Broadcast más tab-oknak
        bc.postMessage({ type: 'FILTER_UPDATE', filters });
      }
    }),
    { name: 'spaceos_catalog', version: 2 }
  )
);
```

**Innováció:** BroadcastChannel API → ha egy tab-ban változik a filter, az összes többi tab real-time frissül. Multi-monitor setup-ban kritikus.

**Kockázat:** localStorage quota (5-10MB) → compression with LZ-string ha túl sok filter preset van.

## 3. Assembly Drag-Drop — Gesztúra-vezérelt + Undo support

### Backend: Operation sequence PATCH
```
PATCH /api/work-orders/{id}/assembly/reorder
Body: { operations: [{ id: 5, new_sequence: 1 }, ...] }
Response: { updated_operations: [...], estimated_duration_change: "+15min" }
```

**Kritikus:** Transaction + validation → ne lehessen logikailag lehetetlen sorrend (pl. "festés" a "csiszolás" előtt).

### Frontend: Gesture-first mobile UX
```jsx
// assembly-operations-list.jsx
import { DndContext, useSensor, PointerSensor, KeyboardSensor } from '@dnd-kit/core';

const sensors = [
  useSensor(PointerSensor, { activationConstraint: { distance: 8 } }),
  useSensor(KeyboardSensor) // A11y: nyíl billentyűkkel is átrendezés
];

<DndContext sensors={sensors} onDragEnd={handleReorder}>
  <SortableContext items={operations}>
    {operations.map(op => (
      <SortableOperation 
        key={op.id} 
        operation={op}
        onUndo={() => history.undo()} // Ctrl+Z support
      />
    ))}
  </SortableContext>
</DndContext>
```

**Innováció 1:** Command Pattern + History Stack
```javascript
const commandHistory = [];
function reorderOperations(from, to) {
  const command = {
    execute: () => applyReorder(from, to),
    undo: () => applyReorder(to, from)
  };
  command.execute();
  commandHistory.push(command);
}
```

**Innováció 2:** Haptic feedback mobilon
```javascript
if ('vibrate' in navigator) {
  navigator.vibrate(10); // drag start
  navigator.vibrate([5, 50, 5]); // drop success
}
```
