# SpaceOS Konsenzus Implementációs Terv

## Összefoglalás

Hibrid megközelítés: konzervatív prioritási sorrend (EHS → Assembly → Partner) kombinálva modern technológiai mintákkal. Az első iteráció REST-alapú, izolált feature-ökkel indít a kockázat minimalizálása érdekében, de event-sourcing és offline-first pattern-ekkel felkészítjük a rendszert későbbi skálázásra. Partner Portal prototípus párhuzamosan fut stakeholder validációhoz, de production deployment EHS után történik.

## Elfogadott prioritás sorrend

1. **EHS Baleset-Bejelentés** (Sprint 1-2) — compliance kritikus, izolált, legkisebb kockázat
2. **Assembly Planning Split View** (Sprint 3-4) — építhet EHS offline mintákra, üzleti érték közepes
3. **Partner-Cockpit Toggle** (Sprint 5) — stabil partner modul feltételezése, legkisebb jogi kockázat csúszásnál

**Párhuzamos track:** Partner Portal low-fidelity prototípus (4 nap, Week 1) stakeholder buy-in biztosítására, éles deployment csak EHS után.

## Backend szükségletek (összesített)

### EHS Baleset-Bejelentés
**Endpoint:** `POST /api/ehs/events` (event-sourced, append-only)
```json
{
  "type": "INCIDENT_REPORTED",
  "payload": {
    "reporterId": "uuid",
    "incidentType": "near-miss|injury|property",
    "locationId": "workshop-zone-id",
    "timestamp": "ISO8601",
    "photoS3Key": "string?",
    "description": "string"
  },
  "meta": {
    "deviceId": "string",
    "clientTimestamp": "ISO8601"
  }
}
```

**Adatmodell:** `ehs_events` tábla (immutable), `ehs_incident_snapshots` view (aktuális állapot). EXIF strip backend-en kötelező, S3 presigned URL upload (ne base64).

**Validáció:** Reporter UUID exists, timestamp drift max ±2h, photo <5MB, idempotency key (UUID v4 client).

---

### Assembly Planning
**Endpoint:** `GET /api/work-orders/{id}/assembly-comparison`
```json
{
  "baselinePlan": {
    "snapshotId": "uuid",
    "frozenAt": "ISO8601",
    "phases": [{"name": "cutting", "plannedHours": 8}]
  },
  "currentState": {
    "phases": [{"name": "cutting", "actualHours": 10, "status": "in-progress"}]
  },
  "variance": {
    "delayHours": 2,
    "criticalPathImpacted": true
  }
}
```

**Adatmodell:** `work_order_baselines` tábla (frozen snapshots), manuális "Save as Baseline" trigger első verzióban (later: RFQ approval webhook).

---

### Partner-Cockpit
**Endpoint:** `GET /api/partners/{id}?scope=internal|external` (REST első verzióban)

**Field filtering middleware:**
```js
const scopeFields = {
  external: ['id', 'name', 'orders', 'deliveries', 'qualityScorePublic'],
  internal: ['*'] // minden mező
};
```

**GraphQL migráció:** Q3 2026 roadmap item, ha mobile app confirmed.

## Frontend megközelítés (legjobb elemek)

### EHS Baleset-Bejelentés
**Komponens:** `<IncidentReportWizard />` (3-step modal)
- **State:** Zustand `incidentDraftStore` + localForage persistálás
- **Offline queue:** localForage + retry service (5min poll), NOT Background Sync API (iOS Safari kompatibilitás miatt)
- **Photo:** Capacitor Camera plugin → S3 presigned URL upload → compress 800px max width client-side
- **UX:** FAB (piros), fullscreen mobile takeover, iOS safe-area-inset

---

### Assembly Planning
**Komponens:** `<AssemblyComparisonView />`
- **Desktop:** CSS Grid split (planned | actual), Recharts superimposed bars
- **Mobile:** Swipeable tabs `[Plan|Actual|Variance]` + `react-window` virtualizáció (50+ phases)
- **State:** React Query, 30s polling ha `status !== 'completed'`
- **Fallback:** Ha performance probléma (>1000 phases), spike PixiJS Canvas rendering Q3-ban

---

### Partner-Cockpit
**Komponens:** `<PartnerCockpit />` + segmented control toggle
```jsx
const [viewMode, setViewMode] = useState('internal');
const { data } = useQuery(['partner', id, viewMode], {
  suspense: true // Suspense streaming pattern
});
```
- **Banner:** Partner módban "👁️ A partner ezt látja" figyelmeztetés
- **Warning:** "Ez preview, NEM permission boundary" dokumentáció

## Amit Sonnet-A-tól veszünk át

1. **Kockázatkezelési részletesség:** EXIF strip backend-en, UUID ütközés mitigáció, photo compression limit
2. **Inkrementális prioritás:** EHS-first megközelítés compliance miatt védhető
3. **localForage + Zustand offline minta:** Egyszerűbb mint Background Sync API, jobb iOS támogatás
4. **Manual baseline gomb:** RFQ webhook bizonytalan függőség, ne blokkoljon
5. **`react-window` virtualizáció:** Assembly nagy adatseteknél azonnal szükséges

## Amit Sonnet-B-től veszünk át

1. **Event sourcing EHS-hez:** Immutable audit trail natív compliance support
2. **S3 presigned URL:** Base64 nem skálázódik, S3 direct upload jobb
3. **Suspense streaming UX:** Partner Cockpit loading state elegánsabb
4. **GraphQL roadmap item:** Ha multi-client jön, migráció architectúra készen van
5. **Performance tudatosság:** Canvas rendering spike befogadása ha Recharts nem elég (Q3 decision point)

## Nyitott kérdések a Conductor-nak

1. **EHS Event retention policy:** Meddig tároljunk immutable eventeket? GDPR right-to-forget implikációk?
2. **Assembly baseline freeze trigger:** RFQ approval webhook mikor ready? Interim manual gomb elfogadható?
3. **Partner Portal timeline:** Stakeholder prototípus demo Q2 vége vagy Q3 eleje? Befolyásolja GraphQL decision-t.
4. **Mobile app roadmap:** 2026 H2-ben várható native iOS/Android? Ha igen, GraphQL előrehozása Sprint 4-re.
5. **Performance SLA:** Assembly Planning >1000 phases supportálása kötelező Launch-ra vagy post-MVP?
6. **Photo storage compliance:** S3 bucket encryption + retention policy audit ready?
7. **Offline sync konfliktus kezelés:** Mi történik ha ugyanazt az incidentet 2 eszközről küldik? (UUID elég vagy UI feedback kell?)
