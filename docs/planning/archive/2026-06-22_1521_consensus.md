**State management:**
```javascript
// app-store.jsx
catalog: {
  products: [...],
  editingCell: { productId, field } | null,
  draftValue: string,
  tempProducts: [] // Optimista duplikáció, "TEMP-" prefix ID
}

// Mutations
duplicateProduct(id, overrides) {
  const temp = { ...findProduct(id), id: `TEMP-${uuid()}`, ...overrides }
  this.tempProducts.push(temp)
  
  // Ha van backend:
  api.duplicate(id).then(real => {
    this.products.push(real)
    this.tempProducts = this.tempProducts.filter(t => t.id !== temp.id)
  }).catch(() => {
    this.tempProducts = this.tempProducts.filter(t => t.id !== temp.id)
    toast.error('Duplikálás sikertelen - próbáld újra')
  })
}
```

**UX Features (B tervből átmentve):**
- **Keyboard shortcuts:** `Cmd+D` duplicate, `Enter` save, `Esc` cancel
- **Smart duplicate:** `suggestNextVariant()` - felismeri "60x200" → javasolja "80x200"
- **Auto-focus:** Duplicate után új sor név cellája szerkesztési módba

**Kockázat mitigáció (A tervből):**
- Debounce save: **500ms** (nem 300ms, agresszív users miatt)
- Touch target: min **44px** height, mobil <768px: modal fallback inline edit helyett
- SKU collision: `${original}-COPY-${timestamp}` suffix

**State management:**
```javascript
// useRfqFilters.js (custom hook)
const useRfqFilters = () => {
  // B terv: URL-based state perzisztencia
  const [filters, setFilters] = useQueryParams({ 
    status: 'open', 
    q: '' 
  })
  
  const filtered = useMemo(() => {
    return sim.quotes
      .filter(rfq => filters.status === 'all' || rfq.status === filters.status)
      .filter(rfq => 
        rfq.rfqNumber.toLowerCase().includes(filters.q.toLowerCase()) ||
        rfq.items.some(item => item.name.toLowerCase().includes(filters.q))
      )
  }, [sim.quotes, filters])
  
  const counts = useMemo(() => ({
    open: sim.quotes.filter(q => q.status === 'open').length,
    quoted: sim.quotes.filter(q => q.status === 'quoted').length,
    closed: sim.quotes.filter(q => q.status === 'closed').length,
    all: sim.quotes.length
  }), [sim.quotes])
  
  return { filtered, filters, setFilters, counts }
}
```

## Kockázatok és mitigációk (konszolidált)

| Kockázat | Súlyosság | Mitigáció | Tulajdonos |
|----------|-----------|-----------|------------|
| Inline edit race condition | KÖZEPES | 500ms debounce + disable input save közben + timestamp conflict resolution | TOP 2 dev |
| Duplikált SKU collision | ALACSONY | `sku-COPY-${timestamp}` + backend unique constraint (409 Conflict) | Backend + TOP 2 dev |
| Search performance 500+ RFQ | ALACSONY | `useMemo` + 300ms debounce, Sprint 3-ban backend pagination ha kell | TOP 3 dev |
| Mobil inline edit touch target | KÖZEPES | Min 44px height + <768px modal fallback | TOP 2 dev |
| Optimista update rollback UX | KÖZEPES | Toast "Művelet sikertelen" + undo gomb 5s timeout | TOP 2 dev |
| Safari private mode localStorage | ALACSONY | `try-catch` wrapper, fallback: sessionStorage | Közös util |
| Parallel dev merge conflict | ALACSONY | Napi sync meeting, feature branch stratégia | Tech lead |
