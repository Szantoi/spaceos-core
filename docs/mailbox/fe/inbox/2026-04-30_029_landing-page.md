# FE-029 — Landing page (publikus bemutatkozó oldal)

> **Cél:** A `/` route-on egy bemutatkozó landing page jelenjen meg a world kártyák helyett. A world kártyák (`HomeScreen`) csak bejelentkezés után legyenek elérhetők.

## Kontextus

Jelenleg a `/` route a world kártyákat mutatja auth előtt. Gábor kérése: legyen egy nyilvános landing page ami bemutatkozik, és a belépés után jelenjenek meg a kártyák.

**Skill:** `/spaceos-terminal` szerint dolgozz
**Használhatsz sub-agent-eket** ha szükséges

## Feladat

### 1. Landing page (`/`)

Egy szép, egyoldalas bemutatkozó oldal:
- **Hero szekció:** JoineryTech logó/wordmark + egysoros szlogen ("A magyar faipar digitális platformja")
- **Belépés gomb:** szembetűnő CTA → Keycloak login
- **3-4 feature blokk:** rövid ikonos leírás mit tud a platform (Megrendelések, Gyártás, Lapszabászat, Raktár)
- **Footer:** joinerytech.hu, copyright
- **Design:** sötét téma (a meglévő `#0f172a` háttér), professzionális, minimál
- **Responsive:** 375px mobil + desktop

### 2. Route átszervezés

```
/                → LandingPage (publikus, NEM auth)
/w               → HomeScreen (world kártyák, RequireAuth)
/w/dashboard     → DashboardPage (RequireAuth) — és a többi mint eddig
/callback        → CallbackPage
```

Tehát:
- A `HomeScreen` (world kártyák) átkerül a `/w` route-ra, `RequireAuth` mögé
- A `/` route-ra az új `LandingPage` kerül
- Sikeres login után a callback `/w`-re navigáljon (nem `/w/dashboard`-ra)

### 3. LandingPage komponens

```
src/pages/LandingPage.tsx
```

- Ha a user már be van jelentkezve → automatikusan redirect `/w`-re
- Ha nincs → landing megjelenik
- "Belépés" gomb → `login()` hívás (useAuth hook)

## Szabályok

- A meglévő oldalak (Dashboard, Orders, stb.) NE változzanak
- A meglévő tesztek zöldek maradjanak
- Magyar feliratok
- Nincs `dangerouslySetInnerHTML`
