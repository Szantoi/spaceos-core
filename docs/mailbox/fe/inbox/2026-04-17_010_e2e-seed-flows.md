## Fontos implementációs irányelvek

**global-setup.ts frissítés:** Az INFRA-139 deploy után a `doorstar-cutting-ready-v1` reset hívható. A global-setup-ban maradhat `empty-v1` — az egyes flow-k elején `beforeEach`-ben hívd a reset endpointot a megfelelő profillal:

```typescript
// tests/e2e/flows/02-cutting-dashboard.spec.ts
import { resetTenant } from '../helpers/resetTenant';

test.beforeEach(async () => {
  await resetTenant('doorstar-cutting-ready-v1');
});
```

**`resetTenant` helper** (add hozzá `tests/e2e/helpers/resetTenant.ts`):
```typescript
export async function resetTenant(seedProfile: string) {
  const { tenantId, apiBase } = testConfig;
  const res = await fetch(
    `${apiBase}/bff/test/tenants/${tenantId}/reset?confirm=true`,
    {
      method: 'DELETE',
      headers: {
        'X-SpaceOS-Internal': 'true',
        'X-SpaceOS-Brand': 'joinerytech',
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ seedProfile }),
    }
  );
  if (!res.ok) throw new Error(`Reset failed: ${res.status}`);
}
```

**Ha egy oldal nem létezik még a portálban:** Ne hagyj failingelő tesztet — használj soft assertion-t vagy `test.skip`-et kommenttel: `// TODO: FE-011 után aktiválni`.
