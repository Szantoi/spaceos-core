# Stock summary endpoint hiányzik — `GET /bff/inventory/stock` (BUG-003b)

## Tünet

`GET /bff/inventory/stock?materialType=MDF18mm` → **404 Not Found**

Mindkét helyen megjelenik:
- Dashboard betöltésekor (stat kártya: "MDF 18mm készlet")
- Inventory / Készlet tab megnyitásakor (pill szűrők)
- Sikeres bevételezés után is — a lista nem frissül (mert a GET 404-et ad)

## Vizuális hatás

| Elem | Jelenlegi | Várt |
|---|---|---|
| Készlet tab Lapok száma | üres / 0 | bevételezett lapok száma |
| Készlet tab Terület | 0.00 m² | bevételezett terület |
| Dashboard stat kártya | 0 | aktuális készlet |

## Feladat

1. Ellenőrizd létezik-e `GET /api/inventory/stock` endpoint az Inventory modulban
2. Ha nem létezik — implementáld:
   - Query param: `materialType` (pl. `MDF18mm`)
   - Response: `{ materialType, totalSheets, totalAreaM2 }` (összesített készlet az adott anyagtípusra)
   - RLS-sel védve (tenant szűrés)
3. Ha létezik — a BFF proxy route hiányzik (ORCH-nak kell jelezni)

## MaterialType értékek (UI-ból)

| UI pill | Query param |
|---|---|
| MDF 18mm | `MDF18mm` |
| MDF 16mm | `MDF16mm` |
| HDF 3mm | `HDF3mm` |
| Forgácslap | `Forgácslap` |
| ABS él | `ABSél` |
