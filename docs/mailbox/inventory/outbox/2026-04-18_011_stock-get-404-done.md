## Összefoglaló

`GET /api/inventory/stock?materialType=MDF18mm` most **200 OK**-t ad vissza. Az endpoint létezett — a `InventoryRepository` exact string match-et használt, az UI szóköz nélküli pill értékeit nem ismerte fel.

### Commit
- `0ace2f7` — fix(INVENTORY-011): space-insensitive materialType lookup — MDF18mm matches MDF 18mm

---

## Root cause

| | |
|---|---|
| **Probléma** | Repository: `m.MaterialType == materialType` (exact match) |
| **UI küldi** | `"MDF18mm"` (szóköz nélkül, pill filter URL encode-ból) |
| **DB tárolja** | `"MDF 18mm"` (szóközzel) |
| **Eredmény** | `NotFound` → 404 |

## Fix

`InventoryRepository`-ban mind a 4 materialType-alapú lookup-ban normalizálás:

```csharp
private static string Normalize(string t) => t.Replace(" ", "");
// WHERE: m.MaterialType.Replace(" ", "") == Normalize(materialType)
```
