# Phase 3C — Migration 0024: `BrandSkinId` + JWT claim

A Phase 3C brand skin rendszerhez a Kernel két kis változást kap:
1. `Tenants` tábla kap egy `BrandSkinId` oszlopot
2. `TokenCommandHandler` a JWT-be beleteszi a `brand_skin` claim-et

---

## Migration 0024

```sql
-- PostgreSQL virtual DEFAULT — nincs table rewrite
ALTER TABLE "Tenants"
    ADD COLUMN "BrandSkinId" varchar(64) NULL;
```

Migration fájlnév: `20260407190000_Migration_0024_TenantsBrandSkinId.cs`

**Nincs CHECK constraint** — a skin ID-k érvényessége az Orchestrator/Portal oldalon van validálva (`resolveBrand()` registry). Ismeretlen érték → `'joinerytech'` fallback.

**EF Core konfig** (`TenantConfiguration.cs` bővítés):
```csharp
builder.Property(t => t.BrandSkinId)
       .HasMaxLength(64)
       .IsRequired(false);
```

**`Tenant` entitás** bővítés:
```csharp
public string? BrandSkinId { get; private set; }

public void SetBrandSkin(string? skinId)
{
    BrandSkinId = skinId;
}
```

---

## TokenCommandHandler — `brand_skin` JWT claim

A meglévő `TokenCommandHandler`-ben (ahol a JWT claims-ek összeállítódnak), add hozzá:

```csharp
// Meglévő claim-ek mellé:
claims.Add(new Claim("brand_skin", tenant.BrandSkinId ?? "joinerytech"));
```

Ha a tenant `BrandSkinId` null → `"joinerytech"` default (a legtöbb meglévő tenant).
