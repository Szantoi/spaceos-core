### 3. Designer.cs generálása

A `.Designer.cs` az EF snapshot metadata. Mivel az EF nem tudja automatikusan generálni egy már alkalmazott migrationhöz, manuálisan kell létrehozni — **vagy** egy "bundle" approach:

**Option A — Scaffold from DB state (ajánlott):**
```bash
# Ellenőrizd hogy a startup project az InMemory/PostgreSQL providert használja,
# NEM SQLite-ot! (ha Development módban SQLite van, használj --no-build alternatívát)

# Ellenőrizd az aktív provider-t:
grep -r "UseSqlite\|UseNpgsql\|UseInMemory" SpaceOS.Kernel.Api/Program.cs

# Ha az EF PostgreSQL provider-rel fut (Production vagy test konfig):
dotnet ef migrations add Migration_0028_StageRegistry_Placeholder \
  --project SpaceOS.Infrastructure \
  --startup-project SpaceOS.Kernel.Api \
  --context AppDbContext

# Ez létrehoz egy ÚJ migrációt — ennek Up() és Down() metódusa ÜRES kell legyen.
# Ha nem üres (azaz az EF "lát" különbséget DbContext és DB között):
# → Nézd meg mi van a generált Up()-ban és döntsd el:
#   - Ha valóban hiányzó entitás → add hozzá az Up()-hoz (de NE DropTable)
#   - Ha SQLite artifact (TEXT UUID stb.) → ez jelzi hogy a provider rossz

# Ha a placeholder üres, töröld, és csak a ModelSnapshot + eredeti 20260410130000 fájlok kellenek.
```

### 4. AppDbContextModelSnapshot.cs javítása

Ez a fő probléma. A KERNEL-060 által generált ModelSnapshot SQLite típusokkal van tele (`"TEXT"`, `"INTEGER"` stb.) — PostgreSQL-en `"uuid"`, `"jsonb"`, `"timestamp with time zone"` kell.

**Megközelítés — PostgreSQL providerrel való regenerálás:**

```bash
# Győződj meg hogy az EF tool PostgreSQL-t lát:
# appsettings.Development.json → ConnectionStrings__DefaultConnection legyen PostgreSQL

# Ha van appsettings.Test.json PostgreSQL-lel, használd azt:
ASPNETCORE_ENVIRONMENT=Test dotnet ef migrations add SchemaSyncCheck \
  --project SpaceOS.Infrastructure \
  --startup-project SpaceOS.Kernel.Api \
  --context AppDbContext

# Nézd meg a generált Up() tartalmát:
# - Ha üres (0 change): a ModelSnapshot már szinkronban van a DbContext entitásokkal ✓
# - Ha nem üres: a különbség megmutatja mi hiányzik/eltér

# Ha üres → commit, töröld a SchemaSyncCheck migrationt, készen vagyunk
# Ha nem üres → elemzés szükséges (jelents vissza)
```
