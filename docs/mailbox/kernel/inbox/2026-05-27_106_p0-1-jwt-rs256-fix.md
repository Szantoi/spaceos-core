# MSG-KERNEL-106 — P0-1: JWT RS256 Authority fix (Cutting · Inventory · Procurement)

## Kontextus

Az Identity modul implementáció lezárult (63 teszt) — production deploy blokkolója a P0-1.

**Vizsgálat eredménye:**

| Service | Jelenlegi állapot | Probléma |
|---|---|---|
| Kernel | `opts.Authority = ...` + `ValidateIssuer = true` | ✅ Rendben |
| Abstractions | `opts.Authority = ...` + `ValidateAudience = true` | ✅ Rendben |
| Joinery | `opts.Authority` + `ArgumentNullException.ThrowIfNullOrEmpty` | ✅ Rendben |
| **Cutting** | `AddJwtBearer(opts => { opts.MapInboundClaims = false; })` | 🔴 Nincs Authority, nincs ValidateIssuer |
| **Inventory** | `AddJwtBearer(opts => { opts.MapInboundClaims = false; })` | 🔴 Nincs Authority, nincs ValidateIssuer |
| **Procurement** | `AddJwtBearer(opts => { opts.MapInboundClaims = false; })` | 🔴 Nincs Authority, nincs ValidateIssuer |

**Miért kritikus:** Authority nélkül a JwtBearer nem ellenőrzi az aláírást — bármilyen, formailag valid JWT elfogadott. A `tid` claim hamisítható, cross-tenant hozzáférés lehetséges.

## Feladat

### 1. Cutting · Inventory · Procurement Program.cs javítás

Mindhárom service-ben a `AddJwtBearer` részt cseréld le a Joinery/Kernel mintájára:

```csharp
// ELŐTTE (nem biztonságos):
builder.Services.AddAuthentication().AddJwtBearer(opts => { opts.MapInboundClaims = false; });

// UTÁNA (Joinery minta alapján):
var jwtAuthority = builder.Configuration["Jwt:Authority"]
    ?? Environment.GetEnvironmentVariable("JWT_AUTHORITY");
var jwtAudience = builder.Configuration["Jwt:Audience"]
    ?? Environment.GetEnvironmentVariable("JWT_AUDIENCE")
    ?? "kernel-api";

ArgumentNullException.ThrowIfNullOrEmpty(jwtAuthority,
    "Jwt:Authority / JWT_AUTHORITY must be configured");

builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(opts =>
    {
        opts.MapInboundClaims = false;
        opts.Authority = jwtAuthority;
        opts.Audience  = jwtAudience;
        opts.RequireHttpsMetadata = !builder.Environment.IsDevelopment();
        opts.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer   = true,
            ValidateAudience = true,
            ValidateLifetime = true,
            ClockSkew        = TimeSpan.FromSeconds(30),
            NameClaimType    = "preferred_username",
            RoleClaimType    = ClaimTypes.Role,
        };
    });
```

### 2. appsettings.json frissítés (mindhárom service)

```json
{
  "Jwt": {
    "Authority": "",
    "Audience": "kernel-api"
  }
}
```

(Üres — production értéket az `.env` / environment variable adja, startup-fail ha hiányzik.)

### 3. Tesztek

Minden érintett service-ben ellenőrizd:
- Meglévő tesztek zöldek maradnak
- Ha van `ApiFactory` / `WebApplicationFactory`: a tesztben lévő JWT helper RS256-ot használ (vagy `opts.Authority = null` a test env-ben)
- Ha a tesztek `SymmetricSecurityKey`-t használnak: cseréld `RsaSecurityKey`-re (mint a Kernel `JwtTestHelper`)
