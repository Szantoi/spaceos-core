**Változtatás tartalma (mindhárom service azonos):**
- `AddAuthentication().AddJwtBearer(opts => { opts.MapInboundClaims = false; })` → teljes RS256 konfig
- `opts.Authority` = `Jwt:Authority` config → `JWT_AUTHORITY` env var
- `opts.Audience` = `Jwt:Audience` → `JWT_AUDIENCE` → `"kernel-api"` default
- `ValidateIssuer = true`, `ValidateAudience = true`, `ValidateLifetime = true`
- `ClockSkew = 30s`, `NameClaimType = "preferred_username"`, `RoleClaimType = ClaimTypes.Role`
- `RequireHttpsMetadata = !IsDevelopment()`
- Production fail-fast: `ArgumentNullException.ThrowIfNullOrEmpty(jwtAuthority, ...)` ha env="Production"
- Új `appsettings.json` mindháromban: `"Jwt": { "Authority": "", "Audience": "kernel-api" }`

**Megjegyzés:** A Joinery mintát követtem (`if (IsProduction()) ThrowIfNullOrEmpty`) — nem az
inbox minta feltétel nélküli változatát — mert az Inventory `InventoryWebFactory<Program>`-ot
használ teszteléshez, és a `Testing` environmentben a JWT_AUTHORITY nincs beállítva.

## Security review

- **Authority + ValidateIssuer**: minden service ellenőrzi a Keycloak RS256 aláírást — bármilyen formailag valid JWT önmagában nem elegendő
- **Audience validation**: `ValidateAudience = true` — cross-service token replay megakadályozva
- **Lifetime**: `ValidateLifetime = true`, `ClockSkew = 30s` — lejárt tokenek nem elfogadottak
- **Production fail-fast**: ha `JWT_AUTHORITY` hiányzik, a service nem indul el — nem csendben degradál
- **RequireHttpsMetadata**: production-ban true, development-ban false (helyes)
- **MapInboundClaims = false**: megmarad — `tid` claim nem lesz átnevezve
- **Nincs secret a logban**: nem logolunk tokent/kulcsot
