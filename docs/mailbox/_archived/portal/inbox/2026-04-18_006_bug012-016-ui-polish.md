# BUG-012/013/015/016 — UI polish batch (4 bug)

## BUG-016 🔴 HIGH — Mobil nézet (375px): sidebar lefedi a tartalmat

**Komponens:** Sidebar / Layout
**Tünet:** 375×667px viewport-on a sidebar `display: block`, `width: 359px` → lefedi a teljes tartalmat. A `☰` Toggle Menu gomb kattintásra sem reagál.
**Elvárt:** Mobilon sidebar alapból rejtett (drawer pattern), Toggle Menu nyit/csuk.

**Fix:** A Topbar `☰` gomb `onClick` nem változtatja a sidebar visibility state-et. Ellenőrizd a sidebar state management-et és `md:` breakpoint beállítást — Tailwind `hidden md:block` minta.

---

## BUG-012 🟡 MEDIUM — Server error után néma hiba (Szállítók modal)

**Komponens:** `CreateSupplierModal` (procurement)
**Tünet:** `POST /bff/procurement/suppliers → 500` esetén a modal csendben bezárul, hibaüzenet nem jelenik meg.
**Elvárt:** "Nem sikerült a mentés. Próbáld újra." bannerrel visszajelzés — ahogy a Rendeléseknél (BUG-009 fix).

**Fix:** `useMutation` `onError` callback hozzáadása a `CreateSupplierModal`-ban (precedens: `CreatePurchaseOrderModal` 3cf19de fix).

---

## BUG-015 🟡 MEDIUM — Escape billentyű nem zárja a modalt (WCAG)

**Komponens:** Minden modal (pl. "Új szállító")
**Tünet:** Modal nyitva marad Escape után.
**Elvárt:** `useEffect` + `keydown` listener → `Escape` → `onClose()` (WCAG 2.1 AA követelmény).

**Fix:** Globális `useEscapeKey(onClose)` hook, vagy a meglévő Modal wrapper komponensbe `onKeyDown` kezelő.

---

## BUG-013 🟢 LOW — Szállítók: maxLength hiányzik a Név mezőn

**Komponens:** `CreateSupplierModal` → Név `<input>`
**Tünet:** `maxLength = -1` (nincs limit) → 500 karakteres input beküldhető.
**Elvárt:** `maxLength={255}` az input-on + backend 400 esetén szép hibaüzenet.

**Fix:** `<input maxLength={255} />` a Név mezőre. Ha a backend `Name` mező is >255-re korlátoz (EF Core `HasMaxLength(255)`), a `useMutation onError` callback (BUG-012 fix) automatikusan mutatja a 400-as hibát.
