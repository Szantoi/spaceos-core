# FREETIER-FE-004 DONE — Nesting response adapter (BUG-FT-005)

## Változtatások

### 1. `src/api/nestingApi.ts` — `fromApiResponse()` frissítve

Az adapter most kezeli a FREETIER-011 API response formátumot:
- **`sheets[].placements[]`** → `placed_parts[]` mapping (+ legacy `placed_parts`/`placedParts` fallback)
- **`placements[].partName`** → `name` mapping
- **`placements[].widthMm/heightMm`** → `width/height` mapping
- **`sheets[].yieldPercent`** → `utilization_percent` mapping
- **`sheets[].wasteAreaMm2`** → `waste_area_mm2` mapping
- **Top-level:** `yieldPercent` → `total_utilization_percent`, `wasteAreaMm2` → `total_waste_mm2`
- **`sheetsAvailable`** fallback: ha API nem adja, `sheets.length`-et használ
- **`totalCutLengthMm`** fallback: 0 ha nincs

### 2. `src/components/visualization/StatCards.tsx` — graceful fallback

- `total_cut_length_mm` (vágási hossz) stat kártyát eltávolítottam — az API nem adja ezt az adatot
- `sheets_available` === 0 esetén csak a `sheets_used` számot mutatja (nem "1 / 0")

### 3. Belső type-ok változatlanok

A `NestingResult`, `SheetResult`, `PlacedPart` type-ok maradnak snake_case-ben. A konverzió kizárólag az API boundary-n történik (`nestingApi.ts`). Ez biztosítja, hogy a 71 meglévő teszt nem törik el.
