## Összefoglaló

BUG-004 root cause véglegesítve: `DateTime.Kind = Unspecified` → Npgsql `timestamptz` write exception. Javítva `DateTime.SpecifyKind(DateTimeKind.Utc)` hívással.

## Root cause (VPS log alapján)

```
System.ArgumentException: Cannot write DateTime with Kind=Unspecified to PostgreSQL type
'timestamp with time zone', only UTC is supported. (Parameter 'value')
```

A `DateTime.TryParse("2026-04-18")` `Kind=Unspecified` DateTime-ot ad vissza. Az Npgsql provider **nem fogadja el** ezt `timestamp with time zone` (timestamptz) oszlopba íráskor — csak `Kind=Utc` megengedett. A hiba `SaveChangesAsync`-ban dobódik → unhandled exception → 500.

**Miért nem bukott meg a unit teszteken?** Az InMemory EF provider C# objektumokat tárol közvetlenül, nem SQL-ben — a `DateTime.Kind`-et nem ellenőrzi. A VPS PostgreSQL-en viszont Npgsql végzi a típuskonverziót, ami stricten ellenőrzi.

## Fix

**`src/SpaceOS.Modules.Cutting.Api/Endpoints/CuttingEndpoints.cs`**

```csharp
// Előtte:
if (!DateTime.TryParse(request.Date, out var planDate))
    return Results.BadRequest(...);
var command = new CreateDailyCuttingPlanCommand(tenantId, request.Name, planDate, batches);

// Utána:
if (!DateTime.TryParse(request.Date, out var rawDate))
    return Results.BadRequest(...);
var planDate = DateTime.SpecifyKind(rawDate.Date, DateTimeKind.Utc);
var command = new CreateDailyCuttingPlanCommand(tenantId, request.Name, planDate, batches);
```
