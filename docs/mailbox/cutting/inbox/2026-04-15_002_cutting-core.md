# MSG-CUTTING-002 — Cutting Core implementáció

## Kontextus

Az Inventory Core (CUTTING-002) kész — 56/56 teszt, `InventoryProviderAdapter` implementálva.

Most jön a Cutting Core: a szabászati végrehajtás domainje.

## Feladat — CUTTING-003

Implementáld a következő projekteket a solution-be:

### 1. SpaceOS.Modules.Cutting.Domain

Aggregatok:
- **`CuttingSheet`** — Egy rendeléshez tartozó szabászati adatcsomag. **Immutable snapshot** — ha újraszámolják, új keletkezik, a régi megmarad. Tulajdonságok: `TenantId`, `OrderReference` (string — külső ref), `Status` (enum: Draft/Submitted/Completed), `Lines` (CuttingLine collection), `CreatedAt`. Fontos: **nincs public setter**, `Submit()` / `Complete()` domain metódusok, minden mutáció domain eventet dob.
- **`CuttingLine`** — Value object / entity a CuttingSheet-en belül. Tulajdonságok: `PartName`, `MaterialType`, `WidthMm`, `HeightMm`, `ThicknessMm`, `Quantity`, `Notes`.
- **`DailyCuttingPlan`** — Napi szabászterv. Tulajdonságok: `TenantId`, `PlanDate`, `Status` (Draft/Finalized), `Batches` (CuttingBatch collection). `Finalize()` domain metódus.
- **`CuttingBatch`** — Egy anyagtípusból összecsoportosított vágások. Tulajdonságok: `MaterialType`, `ThicknessMm`, `SheetIds` (List<Guid> — melyik CuttingSheet-ek tartoznak ide).
- **`CuttingExecution`** — Végrehajtás tracking. FSM: `Planned → InProgress → Completed / Failed`. Tulajdonságok: `TenantId`, `CuttingSheetId`, `AssignedTo` (string), `StartedAt`, `CompletedAt`, `Status`, `WasteAreaCm2` (mért hulladék).

Domain events: `CuttingSheetSubmittedEvent`, `CuttingExecutionCompletedEvent`, `WasteRecordedEvent`

### 2. SpaceOS.Modules.Cutting.Application

CQRS handlerek (MediatR):
- `SubmitCuttingSheetCommand` → `Result<Guid>` — CuttingSheet regisztrálása
- `GetNestingResultQuery` → `NestingResultResponse` — tábla-kiosztás lekérdezése (Phase 1: egyszerű sorrend, nem optimalizált nesting)
- `GetExecutionStatusQuery` → `ExecutionStatusResponse` — végrehajtás állapota
- `GetWasteReportQuery` → `WasteReportResponse` — hulladék riport (dátumtartomány)
- `CreateDailyCuttingPlanCommand` → `Result<Guid>` — napi terv létrehozása
- `GetDailyCuttingPlanQuery` → `DailyCuttingPlanResponse` — napi terv lekérdezése

**Megjegyzés a nesting-ről:** Phase 1-ben `GetNestingResultQuery` nem végez valódi optimalizációt — visszaadja a CuttingLine-okat csoportosítva anyagtípus szerint. A valódi nesting algoritmus (L0–L4) Phase 2+ scope.

### 3. SpaceOS.Modules.Cutting.Infrastructure

- EF Core 8 DbContext: `CuttingDbContext`
- PostgreSQL séma: `spaceos_cutting`
- Migrations: initial migration
- RLS konfiguráció: `CuttingSheets`, `DailyCuttingPlans`, `CuttingExecutions` — `FORCE ROW LEVEL SECURITY`
- Repository: `ICuttingRepository` → `CuttingRepository`
- **`ICuttingProvider` implementáció**: `CuttingProviderAdapter` — delegál a CQRS handler-ekre (ugyanúgy mint az `InventoryProviderAdapter`)

### 4. SpaceOS.Modules.Cutting.Api

Minimal API (port 5004, prefix `/api/cutting`):

```
POST   /api/cutting/sheets                    SubmitCuttingSheet
GET    /api/cutting/sheets/{id}/nesting       GetNestingResult
GET    /api/cutting/sheets/{id}/status        GetExecutionStatus
GET    /api/cutting/waste                     GetWasteReport (query: from, to)
POST   /api/cutting/plans                     CreateDailyCuttingPlan
GET    /api/cutting/plans/{date}              GetDailyCuttingPlan
```

Minden endpoint: `RequireAuthorization("ManufacturerOnly")`
