# MSG-CUTTING-004 — Nesting L1: Greedy FFD PanelAssignment

## Kontextus

A `GetNestingResultQuery` handler jelenleg placeholder — egyszerű anyagtípus szerinti csoportosítást ad vissza, valódi nesting algoritmus nélkül.

A Phase 1 nesting (L1 — Greedy First Fit Decreasing) az első valódi tábla-kiosztási algoritmus.

## Feladat

### 1. PanelAssignment domain objektum

A `SpaceOS.Modules.Cutting.Domain`-ban hozd létre:

```csharp
// PanelAssignment — egy tábla kiosztási eredménye
public class PanelAssignment
{
    public Guid PanelStockId { get; }          // melyik tábla (Inventory-ból)
    public string MaterialType { get; }
    public int PanelWidthMm { get; }
    public int PanelHeightMm { get; }
    public IReadOnlyList<PlacedPart> PlacedParts { get; }
    public int WasteAreaMm2 { get; }           // felhasználatlan terület
    public decimal UtilizationPercent { get; } // kihasználtság %
}

// PlacedPart — elhelyezett alkatrész a táblán
public record PlacedPart(
    string PartName,
    int X, int Y,           // bal felső sarok pozíció
    int WidthMm, int HeightMm,
    bool IsRotated          // 90°-kal elforgatva?
);
```

### 2. NestingService (Domain Service)

```csharp
public class NestingService
{
    // L1 Greedy First Fit Decreasing Height (FFDH) algoritmus:
    // 1. Rendezd a CuttingLine-okat csökkenő magasság szerint
    // 2. Minden alkatrészhez keresd az első táblát ahol elfér
    //    (a maradék szabad területen, fűrészlap rés figyelembevételével: 4mm)
    // 3. Ha nem fér el sehol: nyiss új táblát (következő PanelStock)
    // 4. Return: List<PanelAssignment>

    public IReadOnlyList<PanelAssignment> ComputeNesting(
        IReadOnlyList<CuttingLineRequest> parts,
        IReadOnlyList<AvailablePanel> panels,  // Inventory-ból lekérdezett
        int sawBladeGapMm = 4
    );
}
```

### 3. GetNestingResultQuery handler frissítése

A meglévő `GetNestingResultQueryHandler`-t frissítsd:
1. Lekérdezi az `IInventoryProvider.GetStockAsync()` és `GetOffcutsAsync()` eredményeit
2. Meghívja a `NestingService.ComputeNesting()`-t
3. Visszaadja a `NestingResultResponse`-t a `PanelAssignment`-ekkel

**Fallback:** Ha az Inventory service nem elérhető → graceful degradation (üres nesting result, log warning).

### 4. Tesztek

Minimum 15 új teszt — **fókusz a nesting algoritmuson**:

**Domain (NestingService unit tesztek):**
- 1 alkatrész, 1 tábla → 1 PanelAssignment, helyes pozíció (0,0)
- 2 alkatrész egymás mellé fér → 1 PanelAssignment, 2 PlacedPart
- 2 alkatrész nem fér el egy táblán → 2 PanelAssignment
- Fűrészlap rés (4mm) figyelembevétele a pozíciókban
- Nagy alkatrész előbb kerül (csökkenő sorrend)
- Forgatás: ha fekvő pozícióban nem fér el, álló pozícióban igen
- Kihasználtság % helyes számítása
- Hulladék terület helyes számítása
- Maradék (Offcut) felhasználása prioritásban az új tábla előtt

**Integration (handler + mock Inventory):**
- GetNestingResult → NestingService hívódik, eredmény visszajön
- Inventory service timeout → graceful degradation, nem dobja

## Algoritmikus megjegyzés

Az L1 FFDH egyszerű, de valódi gyártásban ~85% kihasználtságot ér el. **Nem kell L2 guillotine-cut** most — az Q3 scope. A cél hogy a DailyCuttingPlan nesting eredménye használható legyen.

**Egyszerűsítések amik elfogadhatók Phase 1-ben:**
- Nincs dekorminta irány kezelés (minden alkatrész forgatható)
- Nincs gépkorlát ellenőrzés
- 1D strip packing közelítés (sorok, nem teljes 2D bin packing)
