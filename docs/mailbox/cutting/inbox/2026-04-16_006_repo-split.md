# MSG-CUTTING-006 — Repo szétválasztás: Inventory + Procurement önálló repókba

## Kontextus

Az Inventory és Procurement jelenleg a `spaceos-modules-cutting` repóban él, de önálló
üzleti domainek. Ez architektúrálisan helytelen — a szétválasztás most történik meg.

**⚠️ Ez egy migrációs feladat.** Explicit felhatalmazással írsz az alábbi új repo könyvtárakba:
- `/opt/spaceos/spaceos-modules-inventory/`
- `/opt/spaceos/spaceos-modules-procurement/`

## Lépések

### 1. Inventory repo létrehozása

```bash
cd /opt/spaceos/spaceos-modules-inventory
git init
git checkout -b main
```

Fájlok **mozgatása** (nem másolása) a cutting repóból:
```bash
mv /opt/spaceos/spaceos-modules-cutting/src/SpaceOS.Modules.Inventory.* \
   /opt/spaceos/spaceos-modules-inventory/src/

mv /opt/spaceos/spaceos-modules-cutting/tests/SpaceOS.Modules.Inventory.Tests \
   /opt/spaceos/spaceos-modules-inventory/tests/
```

NuGet.Config létrehozása:
```xml
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <packageSources>
    <add key="LocalInventory" value="/opt/spaceos/spaceos-modules-cutting/nupkg" />
    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
  </packageSources>
</configuration>
```

Solution létrehozása:
```bash
cd /opt/spaceos/spaceos-modules-inventory
dotnet new sln -n SpaceOS.Modules.Inventory
dotnet sln add src/SpaceOS.Modules.Inventory.Contracts/
dotnet sln add src/SpaceOS.Modules.Inventory.Domain/
dotnet sln add src/SpaceOS.Modules.Inventory.Application/
dotnet sln add src/SpaceOS.Modules.Inventory.Infrastructure/
dotnet sln add src/SpaceOS.Modules.Inventory.Api/
dotnet sln add tests/SpaceOS.Modules.Inventory.Tests/
```

### 2. Procurement repo létrehozása

Ugyanígy:
```bash
cd /opt/spaceos/spaceos-modules-procurement
git init
git checkout -b main

mv /opt/spaceos/spaceos-modules-cutting/src/SpaceOS.Modules.Procurement.* \
   /opt/spaceos/spaceos-modules-procurement/src/

mv /opt/spaceos/spaceos-modules-cutting/tests/SpaceOS.Modules.Procurement.Tests \
   /opt/spaceos/spaceos-modules-procurement/tests/
```

NuGet.Config — a procurement az Inventory.Contracts-t NuGet-en keresztül kapja:
```xml
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <packageSources>
    <add key="LocalInventory" value="/opt/spaceos/spaceos-modules-cutting/nupkg" />
    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
  </packageSources>
</configuration>
```

Solution létrehozása és ProjectReference → NuGet csere:

A `SpaceOS.Modules.Procurement.Application.csproj`-ban:
```xml
<!-- VOLT (ProjectReference): -->
<ProjectReference Include="../../src/SpaceOS.Modules.Inventory.Contracts/..." />

<!-- LESZ (NuGet): -->
<PackageReference Include="SpaceOS.Modules.Inventory.Contracts" Version="1.1.0" />
```

### 3. Cutting repo lecsupaszítása

A `spaceos-modules-cutting`-ból töröld az Inventory és Procurement project könyvtárakat
(már mozgatva), majd frissítsd a solution fájlt:

```bash
cd /opt/spaceos/spaceos-modules-cutting
dotnet sln remove src/SpaceOS.Modules.Inventory.*
dotnet sln remove src/SpaceOS.Modules.Procurement.*
dotnet sln remove tests/SpaceOS.Modules.Inventory.Tests/
dotnet sln remove tests/SpaceOS.Modules.Procurement.Tests/
```

A `SpaceOS.Modules.Cutting.Application.csproj`-ban a ProjectReference → NuGet:
```xml
<!-- VOLT: -->
<ProjectReference Include="../../src/SpaceOS.Modules.Inventory.Contracts/..." />

<!-- LESZ: -->
<PackageReference Include="SpaceOS.Modules.Inventory.Contracts" Version="1.1.0" />
```
