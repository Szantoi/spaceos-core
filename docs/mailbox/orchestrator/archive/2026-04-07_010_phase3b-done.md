# Phase 3B Response — Snapshot routes + Proof upload + Security review

## 1. BFF route-ok implementálva?

**Igen**

| Fájl | Route-ok |
|------|---------|
| `src/routes/snapshot.route.ts` | `GET /bff/snapshots/:aggregateId`, `GET /bff/snapshots/:aggregateId/versions`, `GET /bff/audit-events/verify-chain` (Admin only) |
| `src/routes/proof.route.ts` | `POST /bff/tasks/:taskId/proof` (streaming, BEFORE body parsing) |

**Verify-chain routing:** `snapshotRouter` az index.ts-ben az `auditEventsProxy` ELŐTT van regisztrálva (`app.use('/bff', snapshotRouter)`), így a `/bff/audit-events/verify-chain` kérés a snapshotRouter-hez kerül, az általános auditEventsProxy nem látja.

## 2. KernelClient bővítve?

**Igen** — `src/kernel/kernelClient.ts`

- `getSnapshot(aggregateId, at)` → GET `/api/snapshots/:id?at=`
- `getSnapshotVersions(aggregateId, page, pageSize)` → GET `/api/snapshots/:id/versions`
- `verifyChain(from, to)` → GET `/api/audit-events/verify-chain`
- `uploadProof(taskId, stream, contentType)` → POST `/api/tasks/:id/proof` (stream-alapú)

Új DTO-k a `src/types/kernel.types.ts`-ben: `SnapshotDto`, `SnapshotVersionDto`, `ChainVerificationDto`, `ProofReceiptDto`.

## 3. Proof upload stream proxy

**Stream (nem bufferelt).** A `proofRouter` az index.ts-ben az `express.json()` ELŐTT van regisztrálva — a request stream intakt marad. Az `axios.post(url, req, { maxBodyLength: Infinity })` közvetlenül pipe-olja a request stream-et a Kernel felé.

## 4. Security review

| Pont | Státusz | Megjegyzés |
|------|---------|-----------|
| UUID validáció (`aggregateId`, `taskId`) | ✅ | UUID regex minden proxy path-ba kerülés előtt, 400 ha invalid |
| `verifyChain` Admin only | ✅ | `requireAdmin` middleware: `roles` array includes `'Admin'` → 403 ha hiányzik |
| `express.json()` nem fut a proof route-on | ✅ | `proofRouter` mount az `express.json()` ELŐTT az index.ts-ben |
| `Authorization: Bearer` header változatlan | ✅ | `kernelHeaders()` helper, JWT nem módosul |
| `X-SpaceOS-Brand` forwarding | ✅ | Minden route-on |
| Content-Type MIME whitelist | ✅ | 415 a BFF-en (`image/*`, `application/pdf`, `video/mp4`, `video/webm`) — Kernel előtt szűr |
