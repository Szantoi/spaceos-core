# MSG-ORCH-082 — Chat 422 fix DONE (+ portal bug azonosítva)

## Root cause

**A 422 ciklus:**

1. A portal `useStreamingChat.ts` a `/bff/chat` (JSON endpoint) endpoint-ot üti POST-tal
2. A sikeres JSON válasz (`{ reply: "...", toolsUsed: [], iterations: 1 }`) nincs `data:` SSE sorral
3. Az SSE olvasó nem talál semmit → az assistant üzenet `content: ''` marad
4. A következő küldéskor `requestMessages` tartalmazza `{ role: 'assistant', content: '' }`
5. Az orchestrátor Zod schema: `z.string().min(1)` → **422 Unprocessable Entity**

Verifikálva a logból: `2026-04-18 08:59:32 POST /bff/chat 422 35.623 ms`

## Orchestrator fix (commit `7b16acb`)

### 1. `src/routes/chat.route.ts` — Schema lazítás
```typescript
// Előtte:
messages: z.array(z.object({
  role: z.enum(['user', 'assistant']),
  content: z.string().min(1),  // üres string is hibát adott
})).min(1)

// Utána: discriminated union
messages: z.array(
  z.discriminatedUnion('role', [
    z.object({ role: z.literal('user'),      content: z.string().min(1) }),
    z.object({ role: z.literal('assistant'), content: z.string() }),  // üres engedett
  ])
).min(1)
```

### 2. `src/interpreter/interpreter.service.ts` — Filter
```typescript
// interpret() és streamChat() is:
const messages = request.messages
  .filter((m) => !(m.role === 'assistant' && m.content === ''))
  .map(...)
```

Üres assistant turn-ök kiszűrve LLM hívás előtt → LLM nem kap confusing empty turnöket.
