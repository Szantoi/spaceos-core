# Skill: Mock API Parallel Development

## Overview

Frontend-independent development using mock API. Eliminates Backend infrastructure blockers (NuGet, build, deploy). Teams can proceed in parallel without waiting.

## When to Use

- Backend has infrastructure blocker (NuGet, network, CI/CD)
- Frontend needs to proceed immediately (time-sensitive deliverables)
- Mock data is realistic enough (20-30 sample entities)
- Real API contracts defined (OpenAPI spec ready)

```typescript
  // ... 14 more opportunities

// Mock API endpoints (simulate delay + error handling)
export const mockCrmApi = {
  async login(email: string, password: string) {
    await delay(300); // Simulate network latency
    return {
      accessToken: 'mock-jwt-token-123',
      refreshToken: 'mock-refresh-token-456',
      expiresIn: 3600
    };
  },

  async getLeads(page = 1, limit = 20) {
    await delay(400);
    return {
      data: mockLeads.slice((page - 1) * limit, page * limit),
      total: mockLeads.length,
      page,
      limit
    };
  },

  async createLead(data: CreateLeadRequest) {
    await delay(200);
    const newLead: LeadDTO = {
      id: Math.random().toString(),
      ...data,
      status: 'new',
      createdAt: new Date().toISOString(),
      updatedAt: new Date().toISOString()
    };
    mockLeads.push(newLead);
    return newLead;
  },

  async updateLeadStatus(id: string, status: LeadStatus) {
    await delay(200);
    const lead = mockLeads.find(l => l.id === id);
    if (!lead) throw new Error('Lead not found');
    lead.status = status;
    lead.updatedAt = new Date().toISOString();
    return lead;
  },

  // ... other endpoints
};

function delay(ms: number) {
  return new Promise(resolve => setTimeout(resolve, ms));
}
```

```typescript
export function connectMockSSE(onMessage: (event: any) => void) {
  // Simulate SSE stream with random events
  const interval = setInterval(() => {
    const events = [
      { type: 'LEAD_CREATED', data: mockLeads[0] },
      { type: 'OPPORTUNITY_UPDATED', data: mockOpportunities[0] }
    ];
    const randomEvent = events[Math.floor(Math.random() * events.length)];
    onMessage(randomEvent);
  }, 5000);

  return () => clearInterval(interval);
}
```

### Phase 3: Integration with TanStack Query

**File:** `src/hooks/useCRM.ts`

```typescript
export const useLeads = (page = 1, limit = 20) => {
  const query = useQuery({
    queryKey: ['leads', page, limit],
    queryFn: () => crmApi.getLeads(page, limit),
    staleTime: 1000 * 60 * 5 // 5 minutes
  });

  return query;
};

export const useCreateLead = () => {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: (data: CreateLeadRequest) => crmApi.createLead(data),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['leads'] });
    }
  });
};
```

## Migration Path: Mock → Real

### Week-by-Week Timeline

| Week | Frontend | Backend | Status |
|------|----------|---------|--------|
| **Week 0** | Uses mock API | OpenAPI spec | Parallel |
| **Week 1** | Wave 2 implementation | Week 2 code | Parallel |
| **Week 2** | Wave 2 testing | Build verification | Parallel |
| **Week 3** | Mock → Real API swap | API endpoints ready | **Integration week** |
| **Week 4** | E2E tests | RLS verification | **Production ready** |

### Cutover Checklist

- [ ] Backend API deployed to staging
- [ ] OpenAPI spec matches implementation (auto-verified by code-gen)
- [ ] JWT token generation + refresh working
- [ ] RLS policies verified (multi-tenant isolation)
- [ ] Frontend feature flag toggled (`VITE_USE_MOCK_API=false`)
- [ ] SSE real events flowing
- [ ] E2E tests passing
- [ ] Production deployment

## Performance Notes

### Mock API Latency

- Simulated: 200-400ms per request
- Real API: Expected similar (network conditions vary)
- No artificial slowness → realistic load testing

### Build Size

- Mock API: +5-10 KB (negligible)
- Real API (Orval-generated): +20-30 KB (type definitions)
- No bloat → production-ready
