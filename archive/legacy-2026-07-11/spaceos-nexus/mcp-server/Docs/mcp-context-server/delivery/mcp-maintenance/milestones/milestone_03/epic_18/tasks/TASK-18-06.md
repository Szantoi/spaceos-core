# TASK-18-06: quality score computation

## Goal

Implement consistent quality scoring for highlights using AI confidence and manual feedback average.

## Affected Files

- `src/metadata/qualityScoring.ts` — NEW
- `src/mcp/AgentDb.ts` — read/update methods for aggregated quality
- `src/tests/unit/highlight-quality-score.test.ts` — NEW

## Acceptance Criteria

Given an existing highlight and optional feedback rows,
When quality score is computed,
Then:

- AC-1: Formula is deterministic and documented.
- AC-2: AI-only case computes valid score in [0.0, 1.0].
- AC-3: AI + feedback case computes expected weighted/average score.
- AC-4: No-feedback case does not fail.
- AC-5: Invalid out-of-range values are clamped or rejected consistently.
- AC-6: Unit tests are green.

## Definition of Done

- [ ] Quality computation service implemented
- [ ] Score update path integrated with highlight persistence
- [ ] Unit tests cover edge cases and boundaries
- [ ] No TypeScript errors
- [ ] Code review approved

## Implementation Hints

- Keep formula versioned in code comments for future migrations.
- Persist both raw components and computed score when practical.
- Avoid floating-point surprises by rounding to fixed precision.

## Risks

- R1: Formula changes breaking comparability over time.
  - Mitigation: add formula version metadata.
- R2: Low-sample feedback bias.
  - Mitigation: apply minimum-count guard before strong feedback weighting.
