# TASK-17-04: switch_domain() + list_available_domains() MCP tool-ok

## Goal

Két új MCP tool implementálása a domain-kontextus kezelésére:

1. `list_available_domains()` — bármely agent lekérdezheti az elérhető domain-eket
2. `switch_domain(domain_name)` — admin/architect agent a session domain-ját váltja

## Affected Files

- `src/mcp/tools/domain-management.ts` — **NEW** plugin/module
- `src/mcp/mcpServer.ts` — modul regisztráció
- `src/tests/unit/domain-management.test.ts` — **NEW**

## Acceptance Criteria

**`list_available_domains()`:**

- AC-1: Visszaadja az összes, `domains` táblában regisztrált domain-t: `{id, name, description}[]`
- AC-2: Ha a `domains` tábla üres, üres listával tér vissza (nem hibázik)
- AC-3: Minden agent role számára elérhető (nincs RBAC restrictor)

**`switch_domain(domain_name)`:**

- AC-4: Csak `admin` vagy `architect` role hívhatja — más role-ok `403 Forbidden` választ kapnak
- AC-5: Ha a megadott `domain_name` nem szerepel a `domains` táblában: `404 Not Found` válasz
- AC-6: Sikeres hívás esetén frissíti a session `current_domain_id` értékét
- AC-7: Sikeres hívás: `{success: true, data: {domain_id, domain_name, previous_domain_id}}`
- AC-8: Audit log: `console.info('[switch_domain] session=... ... → ...')` a domain váltáshoz

**Általános:**

- AC-9: Unit tesztek lefutnak és zöldek

## Definition of Done

- [x] `domain-management.ts` modul implementálva
- [x] Regisztrálva `mcpServer.ts`-ben
- [x] Unit tesztek (AC-1..AC-9)
- [x] Nincsenek TypeScript hibák
- [x] Code review jóváhagyva

## Implementation Hints

- Kövesd a `pm-query.ts` `createPmQueryToolModule(agentDb)` compatibility export mintáját
- RBAC check `isAdminRole()` helper újrahasználható (vagy shared utils-ba kiemelni)
- `switch_domain()` a `SessionManager`-en frissíti a session rekordot (ld. TASK-17-03)
- `list_available_domains()` csak `agentDb.listDomains()` hívás — nincs RBAC check

## Risks

- R1: `switch_domain()` race condition — párhuzamos hívások ugyanarra a sessionre. A `better-sqlite3` single-threaded → SQLite szinten atomikus. Nincs külön lock szükséges.
- R2: RBAC `isAdminRole()` helper duplikálódhat — fontold meg egy shared utility kiemelését (pl. `src/mcp/rbacUtils.ts`), ha már pm-query.ts-ben is létezik
