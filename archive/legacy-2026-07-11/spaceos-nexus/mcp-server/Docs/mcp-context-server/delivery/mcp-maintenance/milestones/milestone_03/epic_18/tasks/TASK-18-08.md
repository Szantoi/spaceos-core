# TASK-18-08: End-to-end reflection cycle

## Goal

Prove the closed-loop learning behavior end-to-end: generated highlights become searchable and are reused by a later session.

## Affected Files

- `src/tests/e2e/epic-18-self-reflection.spec.ts` — NEW
- `src/tests/integration/reflect-session.integration.test.ts` — NEW

## Acceptance Criteria

Given two sequential sessions in the same domain/track,
When Session #1 generates highlights and Session #2 reflects,
Then:

- AC-1: Session #1 stores episode and generated highlight.
- AC-2: Highlight is searchable by keyword (FTS).
- AC-3: Highlight is searchable semantically (ChromaDB path).
- AC-4: Session #2 `reflect_session()` response includes prior lesson references.
- AC-5: Non-regression: existing EPIC-12 flow remains green.
- AC-6: E2E tests are green and deterministic.

## Definition of Done

- [ ] New E2E suite created and stable
- [ ] Integration-level reflection test added
- [ ] Test fixtures deterministic and isolated
- [ ] No TypeScript errors
- [ ] Code review approved

## Implementation Hints

- Use unique temp DB + temp vector namespace per run.
- Keep assertions semantic-safe (contains key phrases, not exact full LLM prose).
- Gate optional external dependencies with graceful fallback in CI.

## Risks

- R1: Flaky E2E due to async indexing race.
  - Mitigation: explicit wait/poll helper for index readiness.
- R2: Environment-dependent vector backend availability.
  - Mitigation: fallback provider and feature-flagged semantic assertions.
