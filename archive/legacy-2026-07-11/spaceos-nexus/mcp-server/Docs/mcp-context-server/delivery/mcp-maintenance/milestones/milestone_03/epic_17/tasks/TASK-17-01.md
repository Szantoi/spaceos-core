# TASK-17-01: Domain schema migration

## Goal

Létrehozni a `domains` táblát a `agent.db`-ben, és hozzáadni a `domain_id` FK-t a `roles` és
`sessions` táblákhoz. Ez az EPIC-17 többi taskjának alapköve.

## Affected Files

- `src/metadata/migrations/008_epic17_domain_schema.sql` — **NEW**
- `src/mcp/AgentDb.ts` — `initSchema()` betölti az új migrációt
- `src/tests/unit/domain-schema.test.ts` — **NEW** unit testek

## Acceptance Criteria

**Given** a fresh `agent.db` (vagy meglévő, migrált),
**When** `AgentDb.initSchema()` lefut,
**Then**:

- AC-1: `domains` tábla létezik az alábbi oszlopokkal: `id TEXT PK`, `name TEXT UNIQUE NOT NULL`, `description TEXT`, `config_json TEXT`, `created_at`, `updated_at`
- AC-2: `roles` táblán `domain_id TEXT` oszlop opcionálisan van (nullable FK), ha a tábla már létezik
- AC-3: `sessions` táblán `current_domain_id TEXT` oszlop nullable FK-ként
- AC-4: Migráció **idempotens**: kétszeri futtatás nem hibázik (IF NOT EXISTS + ALTER TABLE try/catch)
- AC-5: Meglévő adatbázis (`roles` tábla nem üres) esetén a migráció backward-compatible: régi rekordokon `domain_id IS NULL`
- AC-6: Unit teszt lefut és zöld

## Definition of Done

- [x] SQL migráció megírva (`008_epic17_domain_schema.sql`)
- [x] `AgentDb.initSchema()` betölti az új migrációt
- [x] Unit tesztek (AC-1..AC-6)
- [x] Nincs TypeScript compile error
- [x] Code review jóváhagyva

## Implementation Hints

- Kövesd a `007_epic13_blockers_schema.sql` mintát (CREATE TABLE IF NOT EXISTS, index)
- `ALTER TABLE roles ADD COLUMN domain_id TEXT` — try/catch-be csomagolva (SQLite nem engedi, ha már létezik)
- Ugyanez `sessions` táblán (EPIC-08: `sessions` tábla, EPIC-09: `roles` tábla)
- `AgentDb.initSchema()` meglévő migráció-betöltési rendje: 002 → 003 → ... → 008

## Risks

- R1: Ha `roles` schema már más FK-ra épül, `domain_id` nullable kell legyen → AC-5 védjük teszttel
- R2: `sessions` tábla EPIC-08 WriteLayerInitializer inicializálja, `AgentDb` EPIC-09 — koordináció szükséges, melyik inicializáló fut az ALTER TABLE-nel
  - **Megoldás:** az ALTER TABLE-t az AgentDb migration futtatja, a WriteLayerInitializer nem adja hozzá; ez konzisztens az addigi migrációs gyakorlattal (WorkflowStateTracker az agent_sessions-re teszi az ALTER TABLE-t)
