# TASK-17-02: Domain seeder

## Goal

Kiterjeszteni az agent.db seedert úgy, hogy a `database/roles/` könyvtárstruktúra alapján
automatikusan felültöltse a `domains` táblát. Minden alkönyvtár (engineering, management,
discovery stb.) egy domain bejegyzés lesz.

## Affected Files

- `src/mcp/AgentDbSeeder.ts` — kiterjeszteni `seedDomains()` metódussal
- `src/mcp/AgentDb.ts` — új metódusok: `getDomain(name)`, `listDomains()`, `upsertDomain(...)`
- `src/tests/unit/domain-seeder.test.ts` — **NEW**

## Acceptance Criteria

**Given** `database/roles/` tartalmaz `engineering/`, `management/`, `discovery/` könyvtárakat,
**When** `AgentDbSeeder.seed()` lefut,
**Then**:

- AC-1: Minden alkönyvtár (1 szint mélyen) bekerül a `domains` táblába, `name = könyvtárnév`
- AC-2: Seeder idempotens: kétszeri futtatás nem duplikál (`INSERT OR REPLACE`)
- AC-3: `AgentDb.listDomains()` az összes seeded domain-t visszaadja `{id, name, description}[]` formában
- AC-4: `AgentDb.getDomain('engineering')` visszaadja az engineering domain rekordot
- AC-5: Ha a `domains` tábla nem létezik (EPIC-17-01 migration nem futott), a seeder nem crashel — graceful fallback (WARN log, no-op)
- AC-6: Unit teszt lefut és zöld (in-memory DB, mockolt `database/roles/` könyvtár)

## Definition of Done

- [x] `seedDomains()` metódus implementálva
- [x] `listDomains()` / `getDomain()` AgentDb metódusok
- [x] Unit teszt zöld
- [x] Nincs TypeScript hiba
- [x] Code review jóváhagyva

## Implementation Hints

- `fs.readdirSync(databaseRoot + '/roles', {withFileTypes:true}).filter(d => d.isDirectory())` — ez adja a domain-neveket
- Kövesd a meglévő seeder patternjet (`seed-agent-db.test.ts` referencia)
- `upsertDomain(name, description?)` belső helper — `INSERT OR REPLACE INTO domains (name, ...)`
- A `description` mezőt töltsd ki a könyvtár `README.md`-ből (ha van), egyébként `null`

## Risks

- R1: Nem minden `database/roles/` alkönyvtár valódi domain (pl. `_shared/` vagy `adrs/`) — a seeder szűrjön ki `_`-val kezdődő könyvtárakat
- R2: Domain-ek sorrendje nem garantált — TASK-17-05 és 17-06 ne támaszkodjon implicit sorrendre
