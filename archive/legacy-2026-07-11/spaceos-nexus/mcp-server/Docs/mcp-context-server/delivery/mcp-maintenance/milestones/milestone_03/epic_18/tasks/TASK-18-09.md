# TASK-18-09: Reflection cost audit logging

## Goal

Track token and model cost footprint for reflection operations and make it auditable for quality/cost governance.

## Affected Files

- `src/mcp/tools/memory.ts` — emit cost metadata on reflection calls
- `src/metadata/AuditLogger.ts` — extend payload shape (if required)
- `src/tests/unit/reflection-cost-audit.test.ts` — NEW

## Acceptance Criteria

Given reflection tool execution,
When LLM/sampling is invoked,
Then:

- AC-1: Tool captures `ai_model` and `ai_tokens_used` in highlight record.
- AC-2: Audit log records reflection operation with latency and cost fields.
- AC-3: Missing token data does not break tool flow (safe defaults).
- AC-4: Cost metadata is queryable for later reporting.
- AC-5: Unit tests validate logging payload shape and no-crash fallback.
- AC-6: No write-path regression to unrelated tools.

## Definition of Done

- [ ] Cost capture implemented for reflection tools
- [ ] Audit integration completed
- [ ] Unit tests for payload + fallback scenarios
- [ ] No TypeScript errors
- [ ] Code review approved

## Implementation Hints

- Reuse existing audit logging patterns from MCP tool execution path.
- Keep cost schema additive and backward-compatible.
- Store unknown token usage as `NULL` not synthetic values.

## Risks

- R1: Inconsistent cost fields across providers.
  - Mitigation: normalize to common contract at adapter boundary.
- R2: Audit payload bloat.
  - Mitigation: log compact metadata only, avoid full prompt dumps.
