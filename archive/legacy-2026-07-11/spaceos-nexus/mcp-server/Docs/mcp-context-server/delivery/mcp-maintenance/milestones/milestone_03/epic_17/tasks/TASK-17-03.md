# TASK-17-03: bootstrap_agent() — domain context a sessionbe

## Goal

`bootstrap_agent()` tool híváskor az agent`s domain-ja bekerül a sessionbe (`current_domain_id`
mezőként), és ez az érték a `McpContext` részeként visszaadódik minden tool híváskor. Ez teszi
lehetővé a domain-aware RBAC query-ket (TASK-17-05).

## Affected Files

- `src/mcp/tools/bootstrap.ts` — `bootstrap_agent()` handler kiterjesztése
- `src/mcp/SessionManager.ts` — `register()` és `get()` kiterjesztése `current_domain_id`-vel
- `src/mcp/middleware/contextMiddleware.ts` — `McpContext.domain_id` mező hozzáadása
- `src/tests/unit/bootstrap-domain-context.test.ts` — **NEW**

## Acceptance Criteria

**Given** a `bootstrap_agent(domain="engineering", role="backend_developer")` hívás,
**When** a tool sikeresen lefut,
**Then**:

- AC-1: Session rekordba bekerül `current_domain_id` = az engineering domain ID-ja a `domains` táblából
- AC-2: Ha a domain nem szerepel a `domains` táblában, `bootstrap_agent()` fallback: `current_domain_id = null`, WARN log (nem hibázik)
- AC-3: `McpContext.domain_id` tartalmazza a session `current_domain_id` értékét
- AC-4: Meglévő `bootstrap_agent()` tesztek továbbra is zöldek (backward compat — ha domains tábla üres, null-t kap)
- AC-5: `SessionManager.get(sessionId)` visszaadja `current_domain_id`-t a session adatokkal együtt

## Definition of Done

- [x] `bootstrap_agent()` kiterjesztve
- [x] `SessionManager` kiterjesztve
- [x] `McpContext` módosítva
- [x] Unit tesztek zöldek (bootstrap + SessionManager)
- [x] Nincsenek TypeScript hibák
- [x] Code review jóváhagyva

## Implementation Hints

- `McpContext` jelenlegi mezői: `session_id`, `domain`, `role`, `phase`, `track` — adjuk hozzá: `domain_id?: string`
- `SessionManager.register()` jelenleg sqliteben tárolja a session-t (EPIC-08 `sessions` tábla) — hozzáadjuk az `ALTER TABLE` által felvett `current_domain_id` oszlopot
- `bootstrap_agent()` bővítési minta: az AgentDb-től lekérdezi a domain ID-t: `agentDb.getDomain(domain)?.id`
- Ha `getDomain()` null-t ad vissza (domain nem seeded), `current_domain_id = domain` szöveges fallback (EPIC-17 előtti kompatibilitás)

## Risks

- R1: `McpContext` módosítása törhet dolgokat, ha más helyen is kiterjesztik — ellenőrizni a többi `context.*` usage-t (`grep McpContext src/`)
- R2: `sessions` tábla `current_domain_id` oszlopa nullable → a meglévő sessionök `null` értékkel rendelkeznek; test-ekben ez kezelve legyen
- R3: TASK-17-01 és 17-02 szükséges előfeltétel; ha azok nem zöldek, ez a task nem indítható
