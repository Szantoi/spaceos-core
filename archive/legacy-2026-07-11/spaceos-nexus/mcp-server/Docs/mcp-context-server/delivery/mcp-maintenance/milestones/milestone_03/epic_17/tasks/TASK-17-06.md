# TASK-17-06: Multi-domain E2E teszt

## Goal

Komplex end-to-end integráció tesztelése: két domain (engineering + management) párhuzamosan
létezik az `agent.db`-ben, két ügynök session-je egymástól elszigetelten kérdezi le
az adatokat, és a `switch_domain()` tool helyesen váltja a kontextust.

## Affected Files

- `src/tests/integration/multi-domain-e2e.test.ts` — **NEW**

## Acceptance Criteria

**Setup:** engineering + management domain seedelve, mindkettőhöz van `roles` rekord.

**Scenario A — Domain isolation:**

- AC-1: engineering agent `list_available_domains()` → legalább 2 domain látható
- AC-2: engineering agent `request_context()` → visszaad engineering role-t
- AC-3: engineering agent NEM kap management role-adatot (query szigorúan domain_id-re szűr)

**Scenario B — Domain switch:**

- AC-4: Admin agent bootstrap-ol engineering-ben, majd `switch_domain("management")` sikeresen lefut
- AC-5: Váltás után `request_context()` → management kontextust ad
- AC-6: Váltás után `list_my_team_tasks()` → management domain task-ok (ha vannak)

**Scenario C — Cross-contamination védelem:**

- AC-7: Agent-1 engineering session, Agent-2 management session — egyidejű hívás esetén sem kapja a másik agent adatát
- AC-8: `switch_domain()` non-admin agent hívásától 403 Forbidden visszajön

- `switch_domain()` teszteléséhez admin session szükséges — regisztrálj egy harmadik `admin_agent` session-t
- Az izolációt bizonyítani elegendő, ha az engineering agent által lekért role-lista nem tartalmazza management-specifikus role-okat
