# TASK-17-05: RBAC query filtering — domain-aware WHERE clause

## Goal

Az AgentDb query metódusok kibővítése opcionális `domain_id` szűrővel, hogy az agentsek csak
a saját domain-jükre vonatkozó adatokat kapjanak. Ez az EPIC-17 izolációs garanciájának
technikai alapja.

## Affected Files

- `src/mcp/AgentDb.ts` — `getRole()`, `listRoles()`, `getRolesByDomain()`, `getWorkflow()`, `getTemplate()` metódusok kiterjesztése
- `src/mcp/tools/context.ts` — `request_context` tool domain_id-vel szűr
- `src/tests/unit/domain-rbac-filter.test.ts` — **NEW**

## Acceptance Criteria

**Given** `McpContext.domain_id = 'dom-engineering-001'` (az agent engineering domain-ban van),
**When** `list_my_team_tasks()` vagy `request_context()` hívódik,
**Then**:

- AC-1: `AgentDb.listRoles({domain_id: context.domain_id})` csak az engineering domain role-jait adja vissza
- AC-2: Admin role esetén `domain_id = undefined` → szűrés nélkül, minden domain visszajön
- AC-3: Ha `domain_id IS NULL` (legacy adat, TASK-17-01 előtti rekord), a szűrés string-alapú `domain = context.domain` fallback-kel működik
- AC-4: `request_context()` tool: ha a session rendelkezik `domain_id`-vel, az DbQuery ezt használja elsőként; string domain fallback-kel
- AC-5: `pm-query` PM task-listázás (TASK-15): nem törik meg — ha `domain_id` nincs a PM tasks táblájában, a régi `domain` string szűrés érvényesül
- AC-6: Unit tesztek lefutnak: két domain seed → query egyik domain-t → csak annak adatai jönnek vissza

## Definition of Done

- [x] `AgentDb` metódusok kiterjesztve `domain_id` opcionális paraméterrel
- [x] `context.ts request_context` domain_id-vel szűr (ha van)
- [x] Unit tesztek zöldek
- [x] Meglévő `AgentDb.test.ts` tesztek nem törtek (backward compat)
- [x] Nincsenek TypeScript hibák
- [x] Code review jóváhagyva

## Implementation Hints

- Ne touching-olj fel minden single query-t egyszerre — fókuszálj a legfontosabb path-okra: `listRoles`, `getWorkflow`, `getTemplate`
- RBAC query szűrés mintája:

  ```sql
  SELECT * FROM roles WHERE domain = ?
    AND (? IS NULL OR domain_id = ?)
  ```

  Ez backward-compat: ha `domain_id IS NULL` a rekordban → a szűrő figyelmen kívül hagyja
- `AgentDb.listRoles(filters?: {domain?: string, domain_id?: string})` — opcionális param kiterjesztés
- `context.ts request_context` handler a `McpContext`-ből olvassa `domain_id`-t, fallback: `context.domain`

## Risks

- R1: Meglévő tesztek a `domain` string szűrésre támaszkodnak — `domain_id`-s változás ne törje
  - **Mitigáció:** opcionális param, ha `domain_id = undefined` → régi viselkedés
- R2: PM query metódusok (`listPmTasks`) szintén domain-szűrősek — ne duplikáld a logikát; konzisztens filter objektum legyen
