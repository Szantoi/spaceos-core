# TASK-18-07: tag_episode_quality tool

## Goal

Add MCP tool for manual highlight quality feedback and persist it to `highlight_feedback`.

## Affected Files

- `src/mcp/tools/memory.ts` — add `tag_episode_quality`
- `src/mcp/AgentDb.ts` — feedback insert/query helpers
- `src/tests/unit/tag-episode-quality.test.ts` — NEW

## Acceptance Criteria

Given a valid highlight id,
When `tag_episode_quality()` is called,
Then:

- AC-1: Feedback row is created in `highlight_feedback` with score and optional comment.
- AC-2: Score range validation enforces 0.0..1.0.
- AC-3: Unknown `highlight_id` returns standardized `NOT_FOUND`.
- AC-4: Response includes `highlight_id`, `new_quality_score`, `feedback_recorded_at`.
- AC-5: Multiple feedback rows are supported and aggregated.
- AC-6: Unit tests are green.

## Definition of Done

- [ ] Tool and DB helper methods implemented
- [ ] Validation + error handling completed
- [ ] Unit tests cover valid, invalid, and missing highlight scenarios
- [ ] No TypeScript errors
- [ ] Code review approved

## Implementation Hints

- Reuse existing validation style from PM query tools.
- Record `rater_agent_id` from MCP context when available.
- Trigger quality recomputation hook after feedback insert.

## Risks

- R1: Feedback spam skewing score.
  - Mitigation: optional dedupe/rate-limit per rater + session.
- R2: Missing context identity in legacy callers.
  - Mitigation: allow null `rater_agent_id` but keep audit trace.
