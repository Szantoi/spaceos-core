# TASK-17-07: Onboarding dokumentáció

## Goal

Minden jövőbeli fejlesztőnek pontosan értenie kell: hogyan adunk hozzá új domain-t a rendszerhez,
hogyan használja az agent a `switch_domain()` és `list_available_domains()` tool-okat,
és mi történik a háttérben a domain-seeder lefutásakor.

## Affected Files

- `Docs/mcp-context-server/delivery/mcp-maintenance/milestones/milestone_03/epic_17/usage-guide.md` — **NEW**

## Acceptance Criteria

- AC-1: A `usage-guide.md` tartalmazza a "Új domain hozzáadása" step-by-step útmutatót:
  1. Könyvtár létrehozása `database/roles/<domain-name>/` alatt
  2. `seedDomains()` lefuttatása (vagy automatikus server-start triggert dokumentálni)
  3. Ellenőrzés: `list_available_domains()` visszaadja az új domain-t
- AC-2: `switch_domain()` tool használata dokumentálva (ki hivhatja, mi az RBAC feltétel)
- AC-3: `list_available_domains()` dokumentálva (típus, visszatérési mező neve)
- AC-4: Hibakezelés leírva: mi történik, ha nem létező domain-re switchelünk (404 + body)
- AC-5: Mermaid vagy ASCII diagram mutatja a domain-seeder adatfolyamát (`database/roles/` → `AgentDb.seedDomains()` → `domains` tábla → session `current_domain_id`)
