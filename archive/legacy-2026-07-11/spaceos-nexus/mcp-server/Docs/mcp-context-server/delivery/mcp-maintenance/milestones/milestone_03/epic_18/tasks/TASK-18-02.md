# TASK-18-02: generate_episode_highlights

## Goal

Create MCP tool `generate_episode_highlights(session_id)` that reflects recent session episodes and writes normalized highlight records.

## Affected Files

- `src/mcp/tools/memory.ts` — add tool handler
- `src/episodic/EpisodeStore.ts` — helper to load latest episode by session
- `src/mcp/sampling/reflectionPrompts.ts` — NEW prompt contract
- `src/tests/unit/generate-episode-highlights.test.ts` — NEW

## Acceptance Criteria

Given a session with at least one stored episode,
When `generate_episode_highlights(session_id)` is called,
Then:

- AC-1: Tool returns `{ highlight_id, episode_id, key_decisions[], lessons[], next_steps[], quality_score, generated_at }`.
- AC-2: Exactly one highlight row is inserted per `episode_id` (upsert/replace semantics).
- AC-3: If no episode exists for session, tool returns standardized `NOT_FOUND` response.
- AC-4: LLM-assisted path is supported through sampling/reflection prompt integration.
- AC-5: `ai_model` and `ai_tokens_used` are persisted when provided.
- AC-6: Unit tests are green, including malformed LLM output handling.
